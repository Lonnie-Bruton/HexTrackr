Interactive Agent Menu

Purpose & Overview

The interactive-agent-menu.js script provides an interactive interface for an agent memory system. It allows users to:

  1. Continue working on a previous context
  2. Start a new session while keeping previous memories accessible
  3. View a detailed summary of the current context
  4. Search through the agent's memories and knowledge base

This script is a key component of the agent system, enabling users to seamlessly interact with the agent's capabilities and maintain context throughout their development tasks.

Technical Architecture

The InteractiveAgentMenu class is the main entry point for the script. It handles the following responsibilities:

  1. Initialization: Sets up the readline interface for user input/output.
  2. Menu Display: Presents the main menu options to the user and waits for their input.
  3. Input Processing: Handles the user's menu choices and executes the corresponding actions.
  4. Context Management: Interacts with the AgentHelloWorkflow class to initialize the agent, retrieve detailed context, and search through memories.
  5. Error Handling: Catches and displays any errors that occur during the execution.

The script also uses ANSI escape codes to add color and formatting to the console output, making the interface more visually appealing and easy to navigate.

Dependencies

The interactive-agent-menu.js script has the following dependencies:

  1. readline: Node.js built-in module for handling user input/output.
  2. agent-hello-workflow.js: An external module that encapsulates the agent's initialization and memory management logic.

Key Functions/Classes

InteractiveAgentMenu Class

  • Constructor: Initializes the AgentHelloWorkflow instance and the readline interface.
  • start(): Displays the main menu, initializes the agent, and prompts the user for input.
  • promptUser(): Asks the user for their menu choice or custom request.
  • processInput(input): Handles the user's input and executes the corresponding action.
  • showDetailedContext(): Retrieves and displays the detailed context information.
  • searchMemories(): Prompts the user for a search query, executes the search, and displays the results.

AgentHelloWorkflow Class (from agent-hello-workflow.js)

  • initializeAgent(): Initializes the agent and returns the initialization result, including any available context.
  • generateDetailedContext(latestHandoff, personalMemories, mcpMemories, knowledgeDB): Generates a detailed summary of the current context.
  • searchMemories(query): Searches the agent's memories and knowledge base based on the provided query.

Configuration

The interactive-agent-menu.js script does not require any configuration options or environment variables. It relies on the AgentHelloWorkflow class to handle the agent initialization and memory management, which may have its own configuration requirements.

Error Handling

The script handles the following types of errors:

  1. Detailed Context Retrieval Errors: If there is an issue loading the detailed context information, an error message is displayed.
  2. Search Errors: If the search query fails to execute, an error message is displayed.

Any other unexpected errors are caught and displayed to the user with a generic error message.

Integration

The interactive-agent-menu.js script is a key component of the agent system, providing a user-friendly interface for interacting with the agent's capabilities. It is designed to be used as a standalone application or integrated into a larger system that requires an agent-based interface.

Development Notes

  1. ANSI Escape Codes: The script uses ANSI escape codes to add color and formatting to the console output. This helps create a more visually appealing and intuitive interface for the user.
  2. Recursive Prompting: The script uses a recursive approach to prompt the user for input and return to the main menu after each action is completed.
  3. Separation of Concerns: The script separates the menu-related logic from the agent-specific functionality by delegating the agent initialization and memory management to the AgentHelloWorkflow class.
  4. Async/Await: The script makes extensive use of asynchronous functions and the async/await syntax to handle the potentially long-running agent operations.
