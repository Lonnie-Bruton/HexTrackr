Memory Sync Manager for rEngine MCP

Purpose & Overview

The memory-sync-manager.js script is responsible for managing the bidirectional synchronization between a persistent JSON file and the in-memory data store (MCP Memory) for the rEngine MCP (Multimodal Conversational Platform). This ensures that the memory data is resilient against MCP crashes and can be easily restored.

The script provides the following key functionalities:

  1. Load and Save Persistent Memory: Loads the memory data from a JSON file and saves any changes back to the file, with a backup for safety.
  2. Sync to MCP Memory: Attempts to sync the persistent memory data to the MCP Memory, handling any errors or timeouts gracefully.
  3. Pre-commit Sync: Ensures the persistent memory is up-to-date before a Git commit, including an optional merge of Smart Scribe data.
  4. Health Check: Performs a health check on the persistent memory system and updates the memory state accordingly.
  5. Smart Scribe Data Merge: Merges data from the Smart Scribe export into the persistent memory.

This script is a critical component in the rEngine MCP system, ensuring the reliability and persistence of the application's memory data.

Technical Architecture

The MemorySyncManager class is the main entry point for the script. It encapsulates the logic for loading, saving, and synchronizing the persistent memory data.

The class has the following key components:

  1. Persistent Memory File Management: The class maintains the paths for the primary persistent memory file and a backup file.
  2. Load and Save Operations: The loadPersistentMemory() and savePersistentMemory() methods handle the reading and writing of the memory data to the JSON file.
  3. MCP Sync: The syncToMCP() method attempts to sync the persistent memory data to the MCP Memory, with a timeout to prevent the process from hanging.
  4. Pre-commit Sync: The preCommitSync() method coordinates the pre-commit sync process, including the optional merge of Smart Scribe data.
  5. Health Check: The healthCheck() method performs a health check on the persistent memory system and updates the memory state accordingly.
  6. Smart Scribe Data Merge: The mergeSmartScribeData() method merges data from the Smart Scribe export into the persistent memory.

The script also includes a command-line interface (CLI) for manual execution of various operations, such as health checks, backups, and Smart Scribe data merges.
