Memory Health Monitor - Integrated Smart Scribe Module

Purpose & Overview

The memory-health-monitor.js script is a comprehensive tool for real-time monitoring and reporting on the health of critical memory files used by the Smart Scribe system. It performs the following key functions:

  1. File Health Checks: Validates the existence, age, and size of critical memory files to ensure agent continuity.
  2. MCP Connectivity: Checks the accessibility and sync status of the Memory Coordination Plane (MCP) to ensure real-time data availability.
  3. Sync System Health: Verifies the status and timeliness of the automated memory sync process.
  4. Comprehensive Reporting: Generates detailed health reports, including overall status, specific issues found, and recommended actions.
  5. Smart Scribe Integration: Provides a quick health check method for real-time monitoring through the Smart Scribe system.

This script is designed to be a critical component of the larger Smart Scribe system, ensuring the reliability and availability of the memory infrastructure.

Technical Architecture

The MemoryHealthMonitor class is the main entry point for the script, encapsulating all the functionality for checking the health of the memory system. The class has the following key components:

  1. Constructor: Initializes the necessary directories and thresholds for the health checks.
  2. checkFileHealth(): Validates the existence, age, and size of critical memory files.
  3. checkMCPHealth(): Checks the accessibility and sync status of the Memory Coordination Plane (MCP).
  4. checkSyncHealth(): Verifies the status and timeliness of the automated memory sync process.
  5. generateHealthReport(): Consolidates the results of the individual health checks into a comprehensive report.
  6. saveHealthReport(): Saves the health report to a JSON file in the logs directory.
  7. logHealthSummary(): Prints a human-readable summary of the health report to the console.
  8. run(): Executes the full health check process and returns the final report.
  9. quickHealthCheck(): Provides a lightweight health check method for Smart Scribe integration.

The script utilizes several external dependencies, including fs-extra for file system operations, path for managing file paths, and axios for potential future HTTP requests (currently not used).

Usage Examples

To use the MemoryHealthMonitor class, you can create an instance and call the run() method:

const monitor = new MemoryHealthMonitor();
const report = await monitor.run();

This will execute the full health check process and return the comprehensive health report.

For the quick health check method used in Smart Scribe integration, you can call the quickHealthCheck() method:

const quickStatus = await monitor.quickHealthCheck();
console.log(quickStatus);

This will perform a lightweight health check and return an object with the current status.

Configuration

The MemoryHealthMonitor class has the following configuration options:

  1. baseDir: The base directory for the project, default is /Volumes/DATA/GitHub/StackTrackr.
  2. thresholds:
    • maxFileAge: The maximum allowed age for critical memory files (48 hours).
    • maxSyncLag: The maximum allowed lag for the memory sync process (24 hours).
    • minFileSize: The minimum allowed size for critical memory files (100 bytes).
    • maxLogAge: The maximum allowed age for log files (7 days).

These configuration values can be adjusted as needed to fit the specific requirements of your deployment.

Error Handling

The MemoryHealthMonitor class handles errors that may occur during the health check process. If any errors are encountered, they are caught and logged to the console. The run() method will throw the error, allowing the calling code to handle it as needed.

Integration

The MemoryHealthMonitor class is designed to be a central component of the larger Smart Scribe system. It integrates with the system in the following ways:

  1. Real-time Monitoring: The quickHealthCheck() method provides a lightweight health check that can be used by the Smart Scribe system for real-time monitoring and alerting.
  2. Scheduled Validation: The script can be set up to run on a regular schedule (e.g., using cron) to perform comprehensive health checks and update the system's status.
  3. Dashboard Integration: The health reports generated by the script can be used to update the status information displayed on the Smart Scribe dashboard.
  4. Alert Generation: The script can be used to trigger alerts when critical issues are detected, allowing the system to respond proactively to potential problems.

Development Notes

  1. MCP Connectivity Simulation: In the current implementation, the script simulates the MCP connectivity and sync status by reading a local cache file. In a production environment, this would be replaced with the actual MCP integration code.
  2. Extensibility: The script is designed to be extensible, allowing for the addition of new health checks or the modification of existing ones as the requirements of the Smart Scribe system evolve.
  3. Logging and Reporting: The script generates detailed health reports and logs them to the file system. This data can be used for historical analysis and troubleshooting.
  4. Smart Scribe Integration: The quickHealthCheck() method provides a simple, standardized interface for integrating the memory health monitoring functionality into the Smart Scribe system.
