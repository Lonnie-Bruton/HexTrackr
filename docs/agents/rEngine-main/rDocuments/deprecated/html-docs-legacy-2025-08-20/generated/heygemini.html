HeyGemini: Seamless Gemini Testing with OpenWebUI Pipelines

Purpose & Overview

The heygemini.js script is a handy tool for quickly testing the integration between an OpenWebUI instance and the underlying Gemini-powered AI models. It serves as a lightweight wrapper around the OpenWebUI API, providing a simple way to send messages and receive responses from the AI system.

The script is designed to work with the rEngine MCP integration architecture, which powers the OpenWebUI platform. It acts as a bridge between the user's input and the AI models, ensuring a smooth and reliable interaction.

Technical Architecture

The heygemini.js script is built using Node.js and the Axios library for making HTTP requests. It follows a simple, linear flow:

  1. It first attempts to send the user's message to the OpenWebUI Ollama endpoint, which is the primary integration point.
  2. If the Ollama endpoint fails, the script falls back to using the OpenWebUI Pipelines API as a secondary option.
  3. In both cases, the script logs the success or failure of the API calls, and provides helpful troubleshooting information if an error occurs.

The key components of the script are:

  • testOpenWebUI(message): The main function that handles the API calls and response processing.
  • OPENWEBUI_BASE_URL and BACKUP_PIPELINES_URL: The URLs for the primary and secondary API endpoints.
  • OPENWEBUI_API_KEY: The API key required for accessing the OpenWebUI Pipelines API.
