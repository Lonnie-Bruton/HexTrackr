HTML Documentation Generator for rScribe Workers

Purpose & Overview

The html-doc-generator.js script is a powerful tool that converts Markdown files into professional-looking HTML documentation. It was created to streamline the documentation process for rScribe, a platform that manages various worker-related tasks and processes.

The primary purpose of this script is to generate comprehensive, visually appealing, and easily navigable HTML documentation from Markdown source files. This allows rScribe team members and users to access the documentation in a user-friendly format, making it easier to understand and reference the project's technical details.

Technical Architecture

The HTMLDocGenerator class is the main component of this script, responsible for the entire documentation generation process. Here's a breakdown of its key functionality:

  1. Template Loading: The constructor loads a pre-defined HTML template that serves as the base for the generated documentation. This template includes the CSS styles and the navigation structure.
  2. Markdown Conversion: The convertMarkdownToHTML method reads the Markdown content, parses it into HTML using the marked library, and then generates the complete HTML document by combining the parsed content with the navigation and the CSS styles.
  3. Navigation Generation: The generateNavigation method analyzes the Markdown content to extract the section titles and structure, and then generates the navigation sidebar based on this information.
  4. Structured Data Generation: The generateJSON method extracts key metadata from the Markdown content (such as title, sections, word count, etc.) and saves it as a JSON file, providing a structured representation of the documentation.
  5. Multi-Format Generation: The generateMultiFormat method is a convenience wrapper that calls both the convertMarkdownToHTML and generateJSON methods to generate both the HTML and JSON outputs.

The script also includes a command-line interface (CLI) that allows users to generate documentation for a single Markdown file or process multiple files in a batch.

Dependencies

This script relies on the following external dependencies:

  • fs/promises: The Node.js file system module, used for reading and writing files.
  • path: The Node.js path module, used for managing file paths.
  • marked: A Markdown-to-HTML conversion library.

Key Functions/Classes

HTMLDocGenerator Class

This class is responsible for the entire documentation generation process.

Constructor:

  • constructor(): Initializes the class by setting the path to the HTML template file and initializing the CSS and navigation templates.

Methods:

  • loadTemplate(): Reads the HTML template file and extracts the CSS and navigation structures.
  • generateNavigation(sections): Generates the navigation sidebar based on the provided section information.
  • convertMarkdownToHTML(markdownPath, outputPath, options): Reads a Markdown file, converts it to HTML, and writes the result to the specified output path.
  • extractSections(markdownContent): Analyzes the Markdown content and extracts the section structure (titles and sub-titles).
  • generateId(title): Generates a unique ID for a section based on its title.
  • getIconForSection(title): Determines an appropriate icon for a section based on its title.
  • generateCompleteHTML(title, navigation, content): Generates the complete HTML document by combining the title, navigation, and content.
  • generateJSON(markdownPath, jsonPath): Generates a JSON file containing structured data about the documentation.
  • generateMultiFormat(markdownPath): A convenience method that generates both HTML and JSON outputs for a Markdown file.
  • generateHtml(markdownPath): An alias for generateMultiFormat for backward compatibility.

CLI Usage

The script also provides a command-line interface with the following usage:

Usage Examples

  1. Generate HTML Documentation for a Single Markdown File:

    node html-doc-generator.js TASK_SUMMARY.md

    This will generate an HTML file named TASK_SUMMARY.html in the same directory as the Markdown file.

  2. Generate HTML and JSON Documentation for a Single Markdown File:

    node html-doc-generator.js --all SQLITE_MIGRATION_PLAN.md

    This will generate both an HTML file (SQLITE_MIGRATION_PLAN.html) and a JSON file (SQLITE_MIGRATION_PLAN.json) in the same directory as the Markdown file.

  3. Process Multiple Markdown Files in a Batch:

    node html-doc-generator.js --batch *.md

    This will generate HTML and JSON files for all Markdown files in the current directory.

Configuration

The script does not require any specific configuration options or environment variables. However, the HTML template file path is hardcoded in the HTMLDocGenerator constructor, so you may need to update this path if the template file is located in a different location.

Error Handling

The HTMLDocGenerator class handles errors during the following operations:

  1. Template Loading: If the HTML template file cannot be read, an error is thrown with the corresponding error message.
  2. Markdown Conversion: If there is an issue reading the Markdown file or converting it to HTML, an error is thrown with the error message.
  3. JSON Generation: If there is an issue reading the Markdown file or generating the JSON file, an error is thrown with the error message.

In the CLI usage, any errors that occur during the documentation generation process are caught and logged to the console.

Integration

The HTMLDocGenerator class is designed to be used as a standalone tool for generating HTML and JSON documentation from Markdown files. It can be easily integrated into other scripts or processes that require automated documentation generation.

For example, you could integrate this script into a continuous integration (CI) pipeline, where Markdown files are automatically converted to HTML and JSON formats whenever changes are made to the project's documentation.

Development Notes

  1. Markdown Parsing: The script uses the marked library for converting Markdown to HTML. This library is a popular and well-maintained choice for this task, as it provides a simple and robust API.
  2. Navigation Generation: The navigation sidebar is generated dynamically based on the section structure extracted from the Markdown content. This ensures that the navigation accurately reflects the structure of the documentation.
  3. Structured Data Generation: The JSON output provides a structured representation of the documentation, making it easier to integrate with other systems or processes that require access to the documentation metadata.
  4. Customization: While the script provides a default HTML template, it is designed to be easily customizable. You can modify the template file to change the styling, layout, or additional functionality of the generated documentation.
  5. Backward Compatibility: The generateHtml method is provided as an alias for the generateMultiFormat method to maintain backward compatibility with previous versions of the script.

Overall, this script is a powerful tool for automating the generation of professional-looking HTML documentation from Markdown sources, making it a valuable asset for the rScribe project and its users.
