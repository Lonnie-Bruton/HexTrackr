Update Footer Branding Script

Purpose & Overview

The update-footer-branding.js script is responsible for updating the footer branding in all HTML documentation files generated by the rScribe Document System. It ensures that the footer content and styling are standardized across the documentation, providing a consistent and professional appearance.

The script scans the docs/generated/html directory for HTML files, checks if the footer needs to be updated, and then modifies the content and CSS as necessary. This ensures that the documentation maintains a cohesive branding and styling throughout the project.

Technical Architecture

The script follows a modular and asynchronous approach:

  1. File Scanning: The findHTMLFiles function recursively searches the docs/generated/html directory for all HTML files.
  2. File Updating: The updateHTMLFile function is responsible for updating the footer content and CSS in each HTML file.
  3. Main Logic: The main function orchestrates the file scanning and updating process, providing a summary of the changes made.

The script uses the following external dependencies:

  • fs-extra: An enhanced file system module that provides additional functionality over the built-in fs module.
  • path: The built-in path manipulation module.
  • fileURLToPath: A utility function from the built-in url module, used to convert a file URL to a file path.

Key Functions/Classes

findHTMLFiles(dir)

  • Description: Recursively searches the specified directory for all HTML files.
  • Parameters:
    • dir (string): The directory to search.
  • Return Value: An array of file paths for all HTML files found.

updateHTMLFile(filePath)

  • Description: Updates the footer content and CSS in the specified HTML file.
  • Parameters:
    • filePath (string): The file path of the HTML file to update.
  • Return Value: true if the file was updated, false otherwise.

main()

  • Description: The main entry point of the script. It scans the docs/generated/html directory for HTML files, updates the footer in each file, and provides a summary of the changes.
  • Parameters: None.
  • Return Value: None.

Usage Examples

To run the update-footer-branding.js script, execute the following command in your terminal:

node update-footer-branding.js

This will scan the docs/generated/html directory for HTML files and update the footer branding in each file.

Configuration

The script does not require any configuration options or environment variables. It assumes that the docs/generated/html directory exists and contains the HTML files to be updated.

Error Handling

The script handles the following potential errors:

  1. Directory not found: If the docs/generated/html directory does not exist, the script will display an error message and exit.
  2. File update failure: If an error occurs while updating an HTML file, the script will log the error message and continue processing the remaining files.

Integration

This script is designed to be part of the larger rScribe Document System and rEngine ecosystem. It is responsible for ensuring that the generated HTML documentation maintains a consistent branding and styling, which is an important aspect of the overall documentation system.

Development Notes

  1. Supported HTML Patterns: The script currently handles three different footer patterns in the HTML files: <footer class="footer">, <p>Generated from.*?</p>, and <div class="footer">. If the HTML files use a different footer structure, the script may need to be updated to support the new pattern.
  2. Relative Paths: The script uses relative paths when logging file updates to provide a more readable output for the user.
  3. Extensibility: The script can be easily extended to handle additional branding or styling updates in the future, such as updating the header, modifying the overall layout, or customizing the color scheme.
