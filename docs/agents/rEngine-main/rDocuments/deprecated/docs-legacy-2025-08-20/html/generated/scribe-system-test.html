Scribe System Test

Purpose & Overview

The scribe-system-test.js script is a comprehensive test suite for the Scribe system, which consists of two key components:

  1. Smart Scribe (Ollama/Llama): This component is responsible for maintaining and updating the technical knowledge and search optimization databases used by the Scribe system.
  2. Document Generator (Gemini API): This component generates markdown documentation on demand for specific scripts or files.

The purpose of this test script is to ensure that both the Smart Scribe and Document Generator components are functioning correctly, and to provide a way to quickly check the status and outputs of these systems.

Technical Architecture

The scribe-system-test.js script is a Node.js script that uses the following key components:

  1. child_process: The execSync function from the child_process module is used to execute shell commands and capture their output.
  2. fs-extra: The fs-extra module is used to interact with the file system, including checking for the existence of files and directories, and retrieving file metadata.
  3. path: The path module is used to perform operations on file paths, such as joining and relative paths.

The script is divided into three main sections:

  1. Smart Scribe (Ollama/Llama) Test: This section checks the status of the Ollama and Smart Scribe processes, and verifies the existence and metadata of the technical-knowledge.json and search-optimization.json files.
  2. Document Generator (Gemini API) Test: This section tests the Gemini API's document generation functionality by executing the document-generator.js script and verifying the generated documentation files.
  3. Document Sweep Capability: This section checks for the presence of JavaScript files in the rEngine directory, which could potentially be processed by a batch document generation feature.

The script also includes some recommendations based on the test results, suggesting the addition of a batch mode to the document-generator.js script or the creation of a separate script to handle batch document generation.

Dependencies

The scribe-system-test.js script has the following dependencies:

  • child_process: Used for executing shell commands.
  • fs-extra: Used for interacting with the file system.
  • path: Used for performing file path operations.

Key Functions/Classes

The scribe-system-test.js script does not contain any custom functions or classes. It uses the built-in Node.js modules and functions to perform the necessary tasks.

Usage Examples

To run the Scribe System Test, execute the following command in your terminal:

node scribe-system-test.js

This will run the full test suite and display the results in the console.

Configuration

The scribe-system-test.js script has a single configuration option, which is the baseDir variable. This variable is set to the path of the StackTrackr repository directory, and is used to construct the file paths for the various tests.

Error Handling

The script uses try-catch blocks to handle any errors that occur during the execution of the shell commands or file system operations. If an error occurs, the error message is logged to the console.

Integration

The scribe-system-test.js script is designed to test the overall Scribe system, which includes the Smart Scribe (Ollama/Llama) and Document Generator (Gemini API) components. This script is typically used as a standalone utility to ensure the proper functioning of these components within the larger StackTrackr system.

Development Notes

  • The script assumes the existence of certain directories and files within the StackTrackr repository, such as the rEngine directory and the technical-knowledge.json and search-optimization.json files.
  • The script uses the execSync function from the child_process module to execute shell commands, which can be prone to issues if the commands produce a large amount of output or take a long time to execute.
  • The script does not include any comprehensive error handling or logging, and may need to be expanded to provide more detailed and actionable error reporting.
  • The script could be further enhanced by adding support for command-line arguments or configuration files, allowing users to customize the test behavior or target different repositories.

Overall, the scribe-system-test.js script provides a valuable tool for quickly verifying the health and functionality of the Scribe system components within the StackTrackr project.
