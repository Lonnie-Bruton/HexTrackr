add-context.js Documentation

Purpose & Overview

The add-context.js script is an enhanced tool for submitting context entries to a Memory Coordination Protocol (MCP) server. It provides both an interactive mode and a direct command-line interface for users to create new context entries, which can be useful for knowledge management, organizational memory, or other applications that require storing and retrieving contextual information.

The script integrates with the mcp-client.js module to handle the communication with the MCP server, abstracting away the lower-level details and providing a user-friendly interface for submitting new context entries.

Technical Architecture

The add-context.js script is structured as follows:

  1. Entry Point: The main() function serves as the entry point of the script, handling the command-line arguments and launching either the interactive mode or the direct mode based on the provided input.
  2. Interactive Mode: The interactiveMode() function is responsible for managing the interactive session, using the readline module to prompt the user for the necessary information (title, description, and optional entry type) and then submitting the context entry to the MCP server.
  3. Direct Mode: When command-line arguments are provided, the main() function extracts the title, description, and optional entry type from the arguments and calls the addContextEntry() function to submit the context entry to the MCP server.
  4. MCP Integration: The addContextEntry() function is the core of the script, handling the interaction with the mcp-client.js module to submit the context entry to the MCP server. It wraps the addMemory() function from the mcp-client.js module and provides appropriate error handling.
  5. Colorized Output: The script uses ANSI escape codes to add color to the console output, making it more visually appealing and easier to interpret.

Dependencies

The add-context.js script has the following dependencies:

  1. readline: A built-in Node.js module used for the interactive mode to prompt the user for input.
  2. mcp-client.js: An external module that handles the communication with the MCP server, providing the addMemory() function.

Key Functions/Classes

addContextEntry(title, description, entryType = 'general')

  • Description: Submits a new context entry to the MCP server.
  • Parameters:
    • title (string): The title of the context entry.
    • description (string): The description of the context entry.
    • entryType (string, optional): The type of the context entry (default is 'general').
  • Return Value: None.
  • Behavior:
    1. Logs a message indicating that the context entry is being submitted to the MCP server.
    2. Calls the addMemory() function from the mcp-client.js module to submit the context entry.
    3. Logs a success message if the submission is successful.
    4. Logs an error message and exits the process if there is a failure.

interactiveMode()

  • Description: Manages the interactive mode of the script, prompting the user for input and submitting the context entry to the MCP server.
  • Parameters: None.
  • Return Value: None.
  • Behavior:
    1. Creates a readline interface to handle user input.
    2. Prompts the user for the title, description, and optional entry type.
    3. Calls the addContextEntry() function to submit the context entry to the MCP server.
    4. Closes the readline interface when the process is complete.

main()

  • Description: The entry point of the script, handling command-line arguments and launching either the interactive mode or the direct mode.
  • Parameters: None.
  • Return Value: None.
  • Behavior:
    1. Checks the number of command-line arguments provided.
    2. If no arguments are provided, launches the interactiveMode().
    3. If at least two arguments are provided, extracts the title, description, and optional entry type, and calls the addContextEntry() function.
    4. If the arguments are invalid, logs the correct usage and exits the process.

Usage Examples

Interactive Mode

To run the script in interactive mode, simply execute the following command:

node add-context.js

The script will prompt the user for the title, description, and entry type, and then submit the context entry to the MCP server.

Direct Mode

To run the script in direct mode, use the following command-line arguments:

node add-context.js "My Context Entry" "This is a description of the context entry." "custom"

This will submit a context entry with the title "My Context Entry", the description "This is a description of the context entry.", and the entry type "custom" to the MCP server.

Configuration

The add-context.js script does not require any external configuration. All the necessary information (such as the MCP server endpoint) is expected to be handled by the mcp-client.js module.

Error Handling

The add-context.js script uses try-catch blocks to handle errors that may occur during the execution of the addContextEntry() function. If an error occurs, the script will log an error message and exit the process with a non-zero exit code.

Integration

The add-context.js script is designed to be integrated into a larger system that requires the ability to submit context entries to an MCP server. It can be used as a standalone tool or incorporated into other applications that need to manage organizational memory or knowledge.

Development Notes

  • The script uses ANSI escape codes to add color to the console output, which may not be compatible with all terminal environments. Consider using a more cross-platform solution, such as a dedicated logging library, if the script needs to be used in a variety of environments.
  • The script currently only supports submitting context entries to the MCP server. It does not provide any functionality for retrieving or managing existing context entries. If such functionality is required, it should be added to the script or the mcp-client.js module.
  • The script does not handle input validation or data sanitization in depth. In a production environment, additional checks and safeguards should be implemented to ensure the integrity and security of the submitted data.
