test-direct-apis.js - Direct API Integration Test Suite

Purpose & Overview

This script is a utility for testing the direct integration of the Gemini and Groq APIs, bypassing the OpenWebUI layer. It allows developers to quickly validate the functionality and connectivity of these APIs without the need to go through the full OpenWebUI application.

The script performs the following key tasks:

  1. Checks for the required API keys (GEMINI_API_KEY and GROQ_API_KEY) in the environment.
  2. Runs tests for the Gemini and Groq APIs, passing a provided message as input.
  3. Logs the test results, including any errors or successful responses.

This tool is useful for developers who need to troubleshoot API-related issues or validate the integration of these services within their larger application.

Technical Architecture

The script is structured as follows:

  1. Imports: The script imports the necessary dependencies, including the GoogleGenerativeAI class from the @google/generative-ai package and the axios library for making HTTP requests.
  2. API Configuration: The script retrieves the API keys from the environment variables GEMINI_API_KEY and GROQ_API_KEY.
  3. Test Functions: The script defines two asynchronous functions, testGeminiDirect and testGroqDirect, which handle the API testing logic for Gemini and Groq, respectively.
  4. Main Execution: The script takes a message as a command-line argument (or uses a default message if none is provided) and executes the tests for the available APIs (Gemini and/or Groq) based on the presence of the corresponding API keys.

The data flow within the script is as follows:

  1. The script checks the environment for the required API keys.
  2. If the API keys are found, the corresponding test function is called with the provided message.
  3. Inside the test functions, the script creates the necessary API clients and sends requests to the respective APIs.
  4. The API responses are then logged to the console, along with any errors that may have occurred.

Dependencies

The script has the following external dependencies:

  1. @google/generative-ai: A library for interacting with Google's Generative AI services, used for the Gemini API integration.
  2. axios: A popular HTTP client library, used for making requests to the Groq API.

These dependencies are imported at the beginning of the script.

Key Functions/Classes

  1. testGeminiDirect(message):

    • Parameters: message (string) - The message to be passed to the Gemini API.
    • Return Value: void
    • Description: This function handles the integration with the Gemini API. It creates a GoogleGenerativeAI instance using the GEMINI_API_KEY and generates content using the gemini-pro model. The response is then logged to the console.

  2. testGroqDirect(message):

    • Parameters: message (string) - The message to be passed to the Groq API.
    • Return Value: void
    • Description: This function handles the integration with the Groq API. It sends a POST request to the Groq API endpoint using the axios library, with the GROQ_API_KEY provided in the authorization header. The response is then logged to the console.

Usage Examples

To use this script, follow these steps:

  1. Ensure that you have Node.js installed on your system.

  2. Set the required environment variables GEMINI_API_KEY and GROQ_API_KEY with your respective API keys.

  3. Run the script with an optional message argument:

    node test-direct-apis.js "Hello, world!"

    If no message is provided, the script will use a default message: "Hello! Testing direct API integration."

  4. The script will execute the tests for the available APIs (Gemini and/or Groq) and log the results to the console.

Configuration

The script requires the following environment variables to be set:

  • GEMINI_API_KEY: The API key for the Gemini API.
  • GROQ_API_KEY: The API key for the Groq API.

If any of these environment variables are not found, the corresponding API test will be skipped, and a message will be logged to the console.

Error Handling

The script handles errors in the following way:

  1. If the required API keys are not found in the environment, the corresponding API test will be skipped, and an error message will be logged to the console.
  2. If an error occurs during the API requests (e.g., network issues, invalid responses), the error message will be logged to the console.

In both cases, the script will continue to execute the remaining tests (if any) and provide a summary of the test results.

Integration

This script is designed to be a standalone utility for testing the direct integration of the Gemini and Groq APIs. It can be used independently or integrated into a larger application's testing suite to ensure the proper functioning of these API integrations.

Development Notes

  • The script uses the @google/generative-ai library for the Gemini API integration, which provides a convenient wrapper around the Gemini API.
  • The Groq API integration is implemented using the axios library, which allows for straightforward HTTP requests.
  • The script assumes that the API keys are stored in the environment variables. In a production environment, it's recommended to use a more secure method of storing and accessing sensitive information, such as a secrets management service.
  • The script is designed to be simple and easy to understand, focusing on the core functionality of testing the API integrations. In a real-world application, you may want to add additional features, such as more comprehensive error handling, logging, or integration with a testing framework.
