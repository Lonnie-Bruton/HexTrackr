Multi-Provider Rate Limiter

Purpose & Overview

The multi-provider-rate-limiter.js script is a comprehensive rate-limiting solution designed to manage requests to various AI service providers, including Groq, OpenAI, Gemini, and a local provider called Ollama. This script ensures that the rate of requests to these providers is kept within the specified limits, providing a consistent and reliable way to interact with multiple AI services.

Technical Architecture

The MultiProviderRateLimiter class is the core component of this script. It manages the rate-limiting logic for each provider, including:

  • Provider-specific configurations: Each provider has its own set of limits and parameters, such as the maximum tokens per request, requests per minute/hour/day, base delay, and retry settings.
  • Usage tracking: The script keeps track of the number of requests, tokens used, errors, and retries for each provider and time period (minute, hour, day).
  • Request handling: The makeRequest method checks if a request is allowed based on the provider's limits, applies the appropriate delays, and handles retries if necessary.
  • Provider selection: The chooseBestProvider method selects the most suitable provider for a given request based on the provider's limits and current usage.
  • Cleanup and persistence: The script includes a cleanup interval to reset usage counters and methods to save/load usage statistics to/from a file.

The script uses the fs-extra and path modules for file system operations, and the built-in EventEmitter class to emit events for request recording, errors, and retries.

Dependencies

The script has the following dependencies:

  • fs-extra: Provides an enhanced file system API with promises.
  • path: Provides utilities for working with file and directory paths.
  • events: Provides the EventEmitter class for emitting and listening to events.

Usage Examples

import MultiProviderRateLimiter from './multi-provider-rate-limiter.js';

const limiter = new MultiProviderRateLimiter();

// Make a request to Groq
const groqResult = await limiter.makeRequest('groq', () => fetchDataFromGroq(), 1000);

// Make a request to OpenAI
const openaiResult = await limiter.makeRequest('openai', () => generateTextWithOpenAI(), 2048);

// Choose the best provider for a 500-token request
const bestProvider = limiter.chooseBestProvider(500);

Error Handling

The MultiProviderRateLimiter class handles errors in the following ways:

  1. Rate Limit Exceeded: If a request exceeds the provider's rate limits, the script will wait for the appropriate time period (minute, hour, or day) before retrying the request.
  2. Retryable Errors: The script maintains a list of retryable errors, such as "rate limit", "quota exceeded", and various HTTP status codes. If an error occurs, the script will retry the request up to the configured maxRetries limit.
  3. Non-Retryable Errors: If an error is not retryable, the script will log the error and throw it to the caller.

The script emits events for requestRecorded, errorRecorded, and retryRecorded, which can be used to monitor the rate-limiting behavior and errors.

Integration

The MultiProviderRateLimiter class is designed to be integrated into a larger system that requires interaction with multiple AI service providers. By encapsulating the rate-limiting logic, this script can be easily incorporated into a variety of applications, ensuring consistent and reliable access to the AI services.

Development Notes

  • The script uses a conservative approach to setting the default rate limits for each provider, as the actual limits may vary depending on the provider's pricing plan or API tier.
  • The chooseBestProvider method prioritizes providers with higher token limits and lower current usage, but this logic can be further customized based on your specific requirements.
  • The saveUsageStats and loadUsageStats methods provide a way to persist usage statistics, which can be useful for monitoring and billing purposes.
  • The script is designed to be easily extensible, allowing you to add support for new providers or customize the existing provider configurations.

Overall, the multi-provider-rate-limiter.js script provides a robust and flexible solution for managing requests to multiple AI service providers, ensuring that your application can reliably interact with these services while adhering to their rate limits.
