Document Generator

Overview

The document-generator.js script is a powerful tool that automatically generates high-quality documentation for any given JavaScript script. It leverages the capabilities of the Gemini or Groq AI-powered language models to create detailed, markdown-formatted documentation that includes an overview, usage instructions, a breakdown of the core logic, configuration details, and a machine-readable summary.

This script is part of a larger "BrainShare System" that provides various utilities and services to manage and document a software project. By automating the documentation process, this tool helps ensure that all scripts in the system are accompanied by clear, comprehensive, and up-to-date documentation.

How to Use

To generate documentation for a script, run the document-generator.js script from the command line, passing the path to the target script file as an argument:

node document-generator.js /path/to/your-script.js

This will create a new Markdown file in the docs/generated directory, with the same name as the source script but with a .md extension. The generated documentation will be saved to this file.

Example usage:

node document-generator.js ./my-awesome-script.js

Core Logic Breakdown

The DocumentManager class is the main component of this script. It is responsible for the following tasks:

  1. Configuration Loading: The constructor reads the system-config.json file to get the necessary configuration options, such as the API endpoint, model, and provider (Gemini or Groq).
  2. API Key Management: The constructor also retrieves the appropriate API key (either GEMINI_API_KEY or GROQ_API_KEY) from the .env file based on the configured provider.
  3. Documentation Generation: The generate() method is the main entry point for the documentation generation process. It takes the path to the target script file as an argument, reads the file content, and creates a prompt for the language model. It then sends the prompt to the configured API endpoint and saves the generated documentation to a Markdown file.
  4. Prompt Creation: The createPrompt() method constructs the prompt for the language model, which includes instructions for the desired documentation structure and the full script content.
  5. Documentation Saving: The saveDocumentation() method saves the generated documentation to a Markdown file in the docs/generated directory, relative to the script's location.
  6. System Information Update: The updateGeneratedSystemInfo() method updates the generated-system-info.json file with the machine-readable summary extracted from the generated documentation.

The script also includes some error handling and command-line execution logic to handle the case where no file path is provided.

Configuration & Dependencies

The document-generator.js script relies on the following external dependencies:

  • fs-extra: Provides an enhanced file system API with additional utilities.
  • path: Handles file and directory paths.
  • axios: Sends HTTP requests to the Gemini or Groq API.
  • dotenv: Loads environment variables from the .env file.

The script also uses the system-config.json file to retrieve configuration options, such as the API endpoint, model, and provider. This file is expected to be present in the same directory as the document-generator.js script.

The required environment variables are:

  • GEMINI_API_KEY: The API key for the Gemini language model.
  • GROQ_API_KEY: The API key for the Groq language model.

Error Handling

The document-generator.js script includes the following error handling mechanisms:

  1. Missing system-config.json: If the system-config.json file is not found, the script will exit with an error message.
  2. Missing API key: If the required API key (either GEMINI_API_KEY or GROQ_API_KEY) is not found in the .env file, the script will exit with an error message.
  3. File not found: If the provided file path does not exist, the script will display an error message.
  4. API errors: If an error occurs during the API request (either Gemini or Groq), the script will log the error details.
  5. Parsing errors: If the machine-readable summary cannot be parsed from the generated documentation, the script will log a warning message.

Integration

The document-generator.js script is part of a larger "BrainShare System" that provides various utilities and services to manage and document a software project. It is designed to be used as a standalone tool to generate documentation for any JavaScript script, but it can also be integrated into a broader system or workflow.

For example, the document-generator.js script could be incorporated into a continuous integration (CI) pipeline, automatically generating documentation for all scripts in the project whenever changes are made. The generated documentation files could then be committed to the project's repository, ensuring that the documentation is always up-to-date and accessible to the development team.

Development Notes

  1. API Provider Selection: The script supports both Gemini and Groq language models, allowing users to choose the provider that best suits their needs. The provider is determined by the brainShareSystem.documentManager.api.provider configuration in the system-config.json file.

  2. Relative File Paths: The script generates documentation files in the docs/generated directory, relative to the location of the document-generator.js script. This allows the generated documentation to be easily integrated into the project's file structure.

  3. Machine-Readable Summary: The script extracts a machine-readable JSON summary from the generated documentation and updates the generated-system-info.json file. This can be useful for programmatically accessing and processing information about the documented scripts.

  4. Extensibility: The DocumentManager class is designed to be extensible, allowing for potential future improvements or the addition of new features, such as support for additional language models or more customizable documentation templates.

  5. Error Handling Robustness: The script includes thorough error handling to ensure that it can gracefully handle a variety of issues, such as missing configuration files, missing API keys, and API errors. This makes the script more resilient and user-friendly.

  6. Dependency Management: The script uses standard Node.js dependency management techniques, with external dependencies declared in the package.json file and loaded using import statements. This ensures that the script can be easily integrated into other projects or distributed as a standalone utility.
