test-ai-providers.js Documentation

Purpose & Overview

The test-ai-providers.js script is a utility that tests the 5-Tier AI Provider System by sending a sample request and capturing the response. This script is used to validate the functionality of the AI system and ensure that the various providers are working as expected.

Technical Architecture

The script utilizes the child_process module to spawn a new Node.js process that runs the index.js file from the rEngineMCP directory. This child process is used to simulate the AI system and receive the response to the test request.

The key steps are:

  1. Construct a test request object with a predefined set of parameters.
  2. Spawn a child process to run the index.js file.
  3. Listen for the child process's stdout and stderr events to capture the output and error messages.
  4. Send the test request to the child process's stdin.
  5. Wait for the child process to exit and handle the result accordingly.

Dependencies

The script has the following external dependencies:

  • child_process - Node.js built-in module for spawning child processes.

Key Functions/Classes

The main function in the script is testProviders(), which is responsible for the entire test process.

async function testProviders() {
  // ...
}

The function performs the following tasks:

  1. Constructs a test request object with predefined parameters.
  2. Spawns a child process to run the index.js file.
  3. Listens for the child process's stdout and stderr events to capture the output and error messages.
  4. Sends the test request to the child process's stdin.
  5. Waits for the child process to exit and handles the result accordingly.
  6. Returns a Promise that resolves with the AI response or rejects with an error.

Usage Examples

To use the test-ai-providers.js script, follow these steps:

  1. Navigate to the rEngineMCP directory.
  2. Run the script using Node.js:
node test-ai-providers.js

The script will output the AI response to the console, or an error message if the test fails.

Configuration

The script does not have any configuration options or environment variables. The test request parameters are hardcoded in the testRequest object.

Error Handling

The script handles the following errors:

  1. If the child process exits with a non-zero code, the script will log the error output and reject the Promise with an error.
  2. If the test takes longer than 30 seconds, the script will kill the child process and reject the Promise with a timeout error.

Integration

The test-ai-providers.js script is intended to be used as a standalone utility to validate the functionality of the 5-Tier AI Provider System. It can be integrated into a larger testing or deployment pipeline to ensure the AI system is working as expected.

Development Notes

  • The script assumes that the index.js file is located in the /Volumes/DATA/GitHub/StackTrackr/rEngineMCP directory. You may need to update this path if your project is located elsewhere.
  • The test request parameters are hardcoded in the script, but you can modify them to suit your testing needs.
  • The script uses a 30-second timeout to prevent the test from running indefinitely. You may want to adjust this value based on the expected response time of your AI system.
