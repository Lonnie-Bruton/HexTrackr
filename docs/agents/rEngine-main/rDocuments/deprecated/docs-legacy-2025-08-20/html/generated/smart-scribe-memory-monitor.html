Smart Scribe Memory Health Monitor

Purpose & Overview

The smart-scribe-memory-monitor.js script is a comprehensive monitoring tool that integrates with the Smart Scribe application to provide real-time monitoring of the memory health and synchronization status. This script serves the following key purposes:

  1. Real-time Memory Health Checks: Continuously monitors the memory usage and integrity of the Smart Scribe system, including both the MCP (Memory Coordination Platform) server and the local rMemory storage.
  2. MCP Connectivity Validation: Ensures that the MCP server is accessible and running, which is critical for the Smart Scribe application to function properly.
  3. Local Memory File Integrity: Verifies the integrity and staleness of the local memory files, which are essential for agent continuity and data persistence.
  4. Sync Gap Detection: Analyzes the sync status between the MCP server and the local rMemory storage, alerting on any potential sync gaps that could lead to data loss or inconsistencies.
  5. Integration with Smart Scribe: Provides a standardized interface for the Smart Scribe application to access the memory health status, allowing for seamless integration and monitoring within the larger system.

Technical Architecture

The SmartScribeMemoryMonitor class is the main component of this script, and it encapsulates the following key functionality:

  1. Constructor: Initializes the necessary paths and thresholds for the memory health monitoring.
  2. checkMCPConnectivity(): Checks if the MCP server and related processes are running and accessible.
  3. validateLocalMemory(): Validates the integrity and staleness of the critical local memory files.
  4. checkSyncGaps(): Analyzes the sync status between the MCP server and the local rMemory storage.
  5. performHealthCheck(): Orchestrates the various checks and produces a comprehensive health report.
  6. formatForSmartScribe(): Transforms the health report into a format that is compatible with the Smart Scribe application.
  7. getMemoryHealthForScribe(): Provides an integration point for the Smart Scribe application to access the memory health status.
  8. quickConnectivityTest(): Performs a quick connectivity test for the Smart Scribe startup process.

The script also includes a CLI interface for testing and debugging purposes, allowing users to run the various health checks and connectivity tests directly from the command line.

Usage Examples

To use the SmartScribeMemoryMonitor class in your application, you can create an instance and call the various methods:

import SmartScribeMemoryMonitor from './smart-scribe-memory-monitor.js';

const monitor = new SmartScribeMemoryMonitor();

// Perform a comprehensive health check
const healthReport = await monitor.performHealthCheck();
console.log(healthReport);

// Get memory health status for integration with Smart Scribe
const scribeHealth = await monitor.getMemoryHealthForScribe();
console.log(scribeHealth);

// Perform a quick connectivity test
const connectivityTest = await monitor.quickConnectivityTest();
console.log(connectivityTest);

Configuration

The SmartScribeMemoryMonitor class has the following configuration options:

  • baseDir: The base directory where the Smart Scribe application is located (default: /Volumes/DATA/GitHub/StackTrackr).
  • memoryDir: The directory where the rMemory files are stored (default: ${baseDir}/rMemory/rAgentMemories).
  • logDir: The directory where the memory health reports will be saved (default: ${baseDir}/logs).
  • criticalFiles: The list of critical memory files that will be monitored (default: ['memory.json', 'handoff.json', 'tasks.json', 'persistent-memory.json', 'mcp-sync-summary.json']).
  • maxFileAge: The maximum age (in milliseconds) for a memory file to be considered healthy (default: 24 hours).
  • checkInterval: The interval (in milliseconds) at which the comprehensive health check will be performed (default: 15 minutes).

These configuration options can be modified by editing the corresponding properties in the SmartScribeMemoryMonitor constructor.

Error Handling

The SmartScribeMemoryMonitor class handles errors in the following ways:

  1. MCP Connectivity Check: If an error occurs during the MCP connectivity check, the method will return an object with the error property set to the error message.
  2. Local Memory Validation: If an error occurs while reading or validating a local memory file, the validation result will include the error message in the issues array, and the overall health status will be set to 'critical'.
  3. Sync Gap Analysis: If an error occurs during the sync gap analysis, the method will return an object with the status property set to 'error' and the error property set to the error message.
  4. Comprehensive Health Check: If any of the underlying checks encounter an error, the error information will be propagated up to the performHealthCheck() method, and the overall health status will be set accordingly.

In all cases, the error information is captured and made available in the respective method's return value, allowing the calling application to handle the errors appropriately.

Integration

The SmartScribeMemoryMonitor class is designed to integrate seamlessly with the Smart Scribe application. The getMemoryHealthForScribe() method provides a standardized interface for the Smart Scribe application to access the latest memory health status, including a summary, detailed information, and a raw report.

The Smart Scribe application can use this information to:

  1. Display Memory Health Status: Show the current memory health status (healthy, warning, or critical) on the Smart Scribe dashboard.
  2. Trigger Alerts: Generate alerts or notifications when the memory health status changes to a warning or critical state.
  3. Initiate Automated Sync: Trigger an automated memory sync process when a sync gap is detected.
  4. Provide Fallback Capabilities: Use the local memory files as a fallback when the MCP server is unavailable.

By integrating the SmartScribeMemoryMonitor class, the Smart Scribe application can ensure the overall health and reliability of the memory system, providing a seamless experience for users.

Development Notes

  1. Caching and Interval-based Checks: The getMemoryHealthForScribe() method utilizes caching and interval-based checks to optimize performance. If the time since the last health check is less than the configured checkInterval, the method will return the cached health status instead of performing a full check.
  2. Comprehensive Health Reporting: The performHealthCheck() method consolidates the results of the various checks (MCP connectivity, local memory validation, and sync gap analysis) into a comprehensive health report, which includes an overall status and a set of recommendations.
  3. Smart Scribe Integration: The formatForSmartScribe() method transforms the health report into a format that is compatible with the Smart Scribe application, allowing for seamless integration and display of the memory health status.
  4. CLI Interface: The script includes a CLI interface for testing and debugging purposes, allowing users to run the various health checks and connectivity tests directly from the command line.

By following these development notes, you can ensure that the smart-scribe-memory-monitor.js script continues to provide reliable and efficient memory health monitoring for the Smart Scribe application.
