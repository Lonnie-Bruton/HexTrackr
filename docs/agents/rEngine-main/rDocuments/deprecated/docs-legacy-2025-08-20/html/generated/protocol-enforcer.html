Protocol Enforcer

Purpose & Overview

The protocol-enforcer.js script is a critical component of the StackTrackr system, responsible for enforcing strict protocols and security measures to prevent agents from going rogue and breaking established procedures. It acts as a gatekeeper, performing mandatory checks before allowing any significant operations to be executed. This script is designed to integrate with the StackTrackr's memory safety system, ensuring a comprehensive security framework for the entire system.

Technical Architecture

The ProtocolEnforcer class is the core of this script, encapsulating all the protocol enforcement logic. It has the following key components:

  1. Backup Requirement Checks: Ensures that a recent backup (within the last 10 minutes) exists before allowing operations that could potentially modify or restructure critical files or the system.
  2. File Containment Checks: Verifies that all files involved in an operation are within the project's base directory and do not contain any dangerous paths (e.g., the agents/ directory, except for the rAgentMemories/ subdirectory).
  3. Git Operation Checks: Prevents any automated GitHub operations, such as git_push, git_pull, or github_sync, as these must be handled manually by the user.
  4. Memory Consistency Checks: Ensures that all critical memory files (e.g., memory.json, decisions.json, functions.json) exist and are valid JSON.
  5. Agent Behavior Checks: Detects patterns that indicate rogue behavior, such as complete rewrites, deleting all data, ignoring protocols, or bypassing backups. It also requires explicit user confirmation for bulk operations.
  6. Violation Tracking: Keeps track of the number of protocol violations and can initiate an agent lockdown if the maximum number of violations is reached.
  7. Backup Creation: Provides a method to quickly create a protocol backup before risky operations.
  8. Protocol Status Reporting: Exposes the current protocol enforcement status, including whether it's enabled, the number of violations, the maximum allowed violations, and the timestamp of the last backup.

The script also integrates with the MemorySafetySystem class, which is responsible for ensuring the integrity and security of the agent's memory.

Dependencies

The protocol-enforcer.js script imports the following dependencies:

  • fs-extra: Provides a set of file system utilities that extend the native fs module.
  • path: Handles file and directory paths.
  • child_process: Allows executing external commands, such as Git operations.
  • url: Converts file URLs to file paths.
  • memory-safety.js: Imports the MemorySafetySystem class, which is responsible for memory integrity.

Key Functions/Classes

ProtocolEnforcer Class

The ProtocolEnforcer class is the main entry point for the protocol enforcement system.

Constructor

  • Initializes the ProtocolEnforcer instance, including setting the base directory, backup timestamp, and violation tracking.
  • Integrates the MemorySafetySystem instance.
  • Logs the initialization status.

enforceProtocols(operation, details = {})

  • Performs a series of protocol checks for the given operation and details (an object containing additional information, such as a list of files involved).
  • Checks include backup requirements, file containment, Git operations, memory consistency, and agent behavior.
  • Tracks the number of violations and initiates an agent lockdown if the maximum number of violations is reached.
  • Returns an object with the following properties:
    • allowed: Boolean indicating whether the operation is allowed.
    • reason: String explaining the result (e.g., "compliant", "protocol_violations", "max_violations_exceeded").
    • violations: Array of violation objects (if any).
    • lockdown: Boolean indicating whether an agent lockdown has been initiated.

disableProtocols(reason) and enableProtocols()

  • Provides methods to disable and re-enable the protocol enforcement system.
  • Disabling protocols should only be done in emergency situations and with caution.

createProtocolBackup(operation)

  • Creates a Git commit as a protocol backup before a risky operation.
  • The backup message includes the operation name and a timestamp.
  • Updates the lastBackupTime property.

getProtocolStatus()

  • Returns an object with the current protocol enforcement status, including whether it's enabled, the number of violations, the maximum allowed violations, whether a lockdown is in effect, and the timestamp of the last backup.

Other Functions

The script also includes several helper functions that are used within the ProtocolEnforcer class, such as checkBackupRequirement, checkFileContainment, checkGitOperations, checkMemoryConsistency, and checkAgentBehavior.

Usage Examples

To use the ProtocolEnforcer class, you can create an instance and call the enforceProtocols method before executing any significant operations:

import ProtocolEnforcer from './protocol-enforcer.js';

const enforcer = new ProtocolEnforcer();

const operation = 'file_modification';
const details = {
  files: ['path/to/file1.js', 'path/to/file2.js']
};

enforcer.enforceProtocols(operation, details)
  .then(result => {
    if (result.allowed) {
      console.log('Operation approved. Proceeding...');
      // Execute the operation
    } else {
      console.error('Operation blocked by protocol enforcement:', result.reason);
      if (result.lockdown) {
        console.error('Agent locked down due to excessive violations');
      }
    }
  })
  .catch(error => {
    console.error('Protocol enforcement failed:', error);
  });

Configuration

The ProtocolEnforcer class does not have any user-configurable options. However, there are some hardcoded values that can be adjusted if needed:

  • maxViolations: The maximum number of protocol violations allowed before initiating an agent lockdown (default is 3).
  • backupRequired operations: The list of operations that require a recent backup (can be modified as needed).
  • forbiddenGitOps: The list of Git operations that are not allowed to be executed automatically (can be modified as needed).
  • roguePatterns: The list of patterns that indicate rogue agent behavior (can be modified as needed).

Error Handling

The ProtocolEnforcer class handles several types of errors:

  1. Backup Age Check: If the Git log command fails, the backup age is assumed to be infinite, forcing a new backup to be created.
  2. Memory Consistency Checks: If any of the critical memory files are missing or corrupted, the check fails and the violation is reported.
  3. General Errors: Any other errors that occur during the protocol enforcement process are caught and reported.

In all cases, the errors are logged, and the protocol enforcement result reflects the failure, either by returning allowed: false or by throwing an exception.

Integration

The protocol-enforcer.js script is a core component of the StackTrackr system, responsible for enforcing security protocols and preventing agents from going rogue. It integrates closely with the MemorySafetySystem to provide a comprehensive security framework for the entire system.

Development Notes

  1. Backup Requirement: The backup requirement checks are based on the timestamp of the last Git commit, which may not accurately reflect the actual backup state. In a production environment, it may be necessary to implement a more robust backup tracking system.
  2. Git Operations: The script currently only checks for specific Git commands, such as git push. It may be necessary to expand the detection capabilities to cover a wider range of Git operations that could potentially trigger GitHub synchronization.
  3. Memory Consistency: The memory consistency checks assume a specific file structure and naming convention. If the memory management system changes, these checks may need to be updated accordingly.
  4. Rogue Behavior Patterns: The rogue behavior pattern detection is based on a predefined list of patterns. This list may need to be expanded or refined over time as new potential rogue behaviors are identified.
  5. Violation Tracking: The current implementation tracks the number of violations and initiates a lockdown when the maximum is reached. In a production environment, it may be necessary to explore more advanced violation tracking and reporting mechanisms.
  6. CLI Interface: The script includes a basic CLI interface for manual protocol checks. In a production system, this functionality may need to be integrated into a more comprehensive command-line tool or API.

Overall, the protocol-enforcer.js script is a critical component of the StackTrackr system, providing a robust and extensible framework for enforcing security protocols and preventing agent misbehavior.
