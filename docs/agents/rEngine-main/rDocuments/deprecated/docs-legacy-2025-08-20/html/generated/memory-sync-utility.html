Memory Sync Utility

Purpose & Overview

The memory-sync-utility.js script is a critical utility that bridges the gap between the MCP (Memory Control Plane) server's memory and the local rMemory storage. This utility resolves a previously identified 24+ hour sync issue, ensuring data continuity and preventing potential data loss for the agent.

The key features of this utility include:

  1. Pulling Latest MCP Observations: It pulls the latest observations from the MCP and stores them in the local rMemory storage.
  2. Pushing Critical Local Data: It pushes critical local data from the rMemory storage to the MCP memory.
  3. Validating Sync Integrity: It checks the integrity of the sync process and identifies any potential gaps or issues.
  4. Reporting Sync Status: It generates a comprehensive sync report, including details on the sync operations, any identified gaps, and recommendations for further action.

This utility is a priority-0 (P0) critical component, as it ensures data continuity and prevents data loss for the agent.

Technical Architecture

The MemorySyncUtility class is the main component of this script. It performs the following key operations:

  1. Sync to Local: This method pulls the latest MCP data and updates the local rMemory storage, including the memory.json, handoff.json, tasks.json, and persistent-memory.json files. It also creates an MCP sync summary file for local reference.
  2. Validate Sync: This method checks the integrity of the sync process by examining the critical files in the rMemory directory. It identifies any sync gaps, such as missing files or stale data, and generates a validation report.
  3. Generate Sync Report: This method provides a comprehensive summary of the sync operation, including the duration, the number of sync operations, and the next steps for maintaining the sync health.

The script also includes an entry point that creates an instance of the MemorySyncUtility class and runs the sync process.

Dependencies

This script relies on the following external dependencies:

  1. fs-extra: A file system library that provides enhanced functionality over the built-in fs module.
  2. path: A built-in Node.js module for working with file paths.

Usage Examples

To use the MemorySyncUtility, you can create an instance of the class and call the run() method:

import MemorySyncUtility from './memory-sync-utility.js';

const syncUtil = new MemorySyncUtility();
syncUtil.run();

This will execute the full sync process, including pulling the latest MCP data, validating the sync integrity, and generating a sync report.

Configuration

This script does not require any external configuration options or environment variables. The necessary configuration, such as the base directory and critical file paths, is handled within the MemorySyncUtility class.

Error Handling

The MemorySyncUtility class handles errors that may occur during the sync process. If an error is encountered, it will be caught and logged to the console, and the script will exit with a non-zero status code.

Integration

The memory-sync-utility.js script is a critical component of the overall system, as it ensures data continuity and prevents potential data loss for the agent. It is designed to be integrated into the larger system's workflow, potentially running on a regular schedule (e.g., daily) to maintain the sync between the MCP and the local rMemory storage.

Development Notes

  1. Sync Metadata: The script maintains a sync metadata object that tracks the sync direction, timestamp, critical updates, and sync status. This metadata is used to update the persistent-memory.json file and generate the sync log.
  2. MCP Sync Summary: The script creates an mcp-sync-summary.json file in the rMemory directory to provide a local reference for the latest MCP observations and relevant system information.
  3. Validation Report: The validateSync() method generates a detailed validation report, which is written to a file in the logs directory. This report can be used to identify and address any sync gaps or issues.
  4. Recommendations: The validation report includes specific recommendations for addressing any identified sync gaps, such as running the sync more frequently or caching the latest MCP data locally.
  5. Sync Log: The script maintains a sync log, which records the specific operations performed during the sync process. This log is included in the sync report and can be used for debugging and troubleshooting purposes.
