StackTrackr Scribe Console

Purpose & Overview

The scribe-console.js script provides an interactive console interface for managing conversation summaries and agent memory in the StackTrackr system. It allows users to generate conversation summaries, list and read agent memory files, search across memories, and perform various system status checks. The console is designed to be a powerful tool for developers and operators to interact with the StackTrackr AI engine and its associated memory management systems.

Technical Architecture

The script is structured as a ScribeConsole class, which encapsulates the functionality of the interactive console. The class has the following key components:

  1. Readline Interface: The readline module is used to create an interactive command-line interface, handling user input and output.
  2. Memory and Engine Paths: The script maintains references to the paths where agent memory files and the StackTrackr engine components are located.
  3. Logging System: The class provides a custom logging system with different log levels (INFO, SUCCESS, ERROR, VERBOSE, SYSTEM) and corresponding colors for improved readability.
  4. Command Handling: The executeCommand method dispatches and handles various commands entered by the user, such as generating summaries, managing memory, and checking system status.
  5. Summary Generation: The generateSummary method calls the scribe-summary.js script to generate conversation summaries for a specified timeframe.
  6. Memory Management: The handleMemoryCommand method provides sub-commands to list, read, and search agent memory files.
  7. System Status Checks: The showStatus method checks the integrity of the memory system, engine system, and Git repository status.

Dependencies

The script imports the following modules:

  • readline: For creating the interactive command-line interface.
  • fs/promises: For asynchronous file system operations.
  • path: For handling file paths.
  • child_process: For executing the scribe-summary.js script.

Usage Examples

To use the Scribe Console, run the following command in your terminal:

node scribe-console.js

This will start the interactive console, and you can then use the following commands:

  • summary [timeframe]: Generate a conversation summary for the specified timeframe (e.g., summary 6h).
  • memory list: List all available agent memory files.
  • memory read [file]: Read the contents of a specific memory file (e.g., memory read tasks).
  • memory search [term]: Search for a term across all agent memory files.
  • status: Show the current system status, including memory and engine information, as well as Git repository status.
  • verbose on/off: Toggle verbose logging.
  • clear: Clear the console.
  • help: Display the available commands and usage examples.
  • exit: Exit the Scribe Console.

Configuration

The Scribe Console script does not require any explicit configuration. However, it relies on the following directories and files being present in the current working directory:

  • rMemory/rAgentMemories: The directory where agent memory files are stored.
  • rEngine: The directory containing the StackTrackr engine components, including the scribe-summary.js script.

Error Handling

The Scribe Console script handles various errors that may occur during its execution, such as:

  • Memory path or engine path not found
  • Scribe summary system not found
  • Failure to generate a summary
  • Failure to list, read, or search memory files
  • Failure to check system status

In case of an error, the script will log the error message with the appropriate log level (e.g., 'ERROR') and provide additional details in the verbose log.

Integration

The Scribe Console is designed to be a standalone tool that interacts with the StackTrackr AI engine and its associated memory management systems. It can be used by developers, operators, or end-users to manage and analyze the conversational data and agent memory within the StackTrackr system.

Development Notes

  • The script uses ANSI escape codes to add color to the console output, which may not be compatible with all terminal environments.
  • The script assumes that the scribe-summary.js script is available in the rEngine directory and can be executed using node.
  • The script does not provide any persistence or backup mechanisms for the agent memory files. Users should ensure that the memory files are properly managed and backed up as needed.
  • The script does not handle concurrent access to the agent memory files, which may lead to potential issues if multiple instances of the Scribe Console are running simultaneously.
