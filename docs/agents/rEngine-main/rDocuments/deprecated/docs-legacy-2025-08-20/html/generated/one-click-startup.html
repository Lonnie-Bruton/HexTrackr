One-Click Startup: Complete StackTrackr AI System

Purpose & Overview

The one-click-startup.js script is a comprehensive solution that automates the startup and configuration of the StackTrackr AI system. It performs the following key tasks:

  1. Starts MCP Memory Server: Launches the MCP (Model Context Protocol) Memory server with visible notifications, enabling developers to track memory operations.
  2. Launches Smart Scribe: Initializes the Smart Scribe system, which is the Ollama agent responsible for analyzing the codebase.
  3. Enables MCP Widgets and Inline Notifications: Ensures that all MCP-related widgets and inline notifications are enabled, providing developers with real-time visibility into the system's operations.
  4. Creates Git Backup and Enforces Protocol Compliance: Generates a Git checkpoint to ensure protocol compliance and maintain a version-controlled backup of the system.
  5. Shows Live Status with Confidence Indicators: Displays a comprehensive dashboard with information about running processes, protocol compliance status, and memory system readiness.

This script aims to provide a seamless and hassle-free startup experience for the StackTrackr AI system, enabling developers to quickly set up the environment and start working with the system.

Technical Architecture

The one-click-startup.js script is built using Node.js and several core Node.js modules, including child_process, util, fs-extra, and path. The script is organized into the following key components:

  1. OneClickStartup Class: This class encapsulates the entire startup process and provides methods for each of the key tasks, such as checking Docker requirements, starting the MCP Memory server, launching the Smart Scribe system, and enabling inline notifications.
  2. Startup Sequence: The start() method in the OneClickStartup class orchestrates the entire startup sequence, executing the various tasks in a specific order.
  3. Utility Functions: The script utilizes several utility functions, such as execPromise (a promisified version of exec from the child_process module) and a set of ANSI color codes for formatted console output.

The script uses a combination of spawning child processes (for running external commands) and executing inline JavaScript operations (for tasks like writing files and enabling inline notifications) to achieve the desired functionality.

Dependencies

The one-click-startup.js script relies on the following external dependencies:

  1. @modelcontextprotocol/server-memory: The MCP Memory server package, responsible for managing the system's memory and knowledge base.
  2. fs-extra: An enhanced file system module that provides additional functionality over the built-in fs module.
  3. Docker: The script assumes the presence of Docker Desktop and performs checks to ensure the Docker requirements are met.

Key Functions/Classes

OneClickStartup Class

Constructor:

  • Initializes the class with the base directory for the StackTrackr project and the directories for the rEngine and other relevant components.
  • Sets the default values for MCP widget visibility and inline notifications.

checkDockerRequirement():

  • Checks if Docker is installed and running on the system.
  • Attempts to run a custom Docker requirement check script, and if that fails, falls back to a basic Docker version and info check.

  • Orchestrates the entire startup sequence, including Git backup, MCP Memory server start, Smart Scribe launch, Scribe console initialization, inline notification enablement, system visibility testing, and live dashboard display.

launchScribeConsole():

  • Opens a new Terminal window and runs the Scribe console, which displays live memory operations.

enableInlineNotifications():

  • Enhances the memory writer functionality to show inline notifications for every memory operation.

  • Performs a series of sample memory write and read operations to verify the visibility of the system.

showLiveDashboard():

  • Displays a comprehensive dashboard with information about running processes, protocol compliance status, and memory system readiness.

Usage Examples

To use the one-click-startup.js script, follow these steps:

  1. Ensure that you have Node.js and Docker Desktop installed on your system.
  2. Navigate to the StackTrackr project directory in your terminal.
  3. Run the script using the following command:
node one-click-startup.js

Configuration

The OneClickStartup class has a few configuration options that can be adjusted:

  1. baseDir: The base directory for the StackTrackr project, which is set to /Volumes/DATA/GitHub/StackTrackr by default.
  2. engineDir: The directory for the rEngine component, which is set to path.join(this.baseDir, 'rEngine').
  3. mcpVisible: A boolean flag that controls whether the MCP widgets are visible (set to true by default).
  4. inlineNotifications: A boolean flag that controls whether inline notifications are enabled for memory operations (set to true by default).

These configuration options can be modified within the OneClickStartup constructor to adjust the startup behavior as needed.

Error Handling

The one-click-startup.js script handles errors at various stages of the startup process. If any of the critical tasks fail, the script will log an error message and exit with a non-zero status code. Some examples of error handling:

  • If Docker requirements are not met, the script will log an error message and exit.
  • If there are any issues with starting the MCP Memory server, Smart Scribe, or launching the Scribe console, the script will log an error message and continue with the next step.
  • If the system visibility test or live dashboard display encounters any problems, the script will log a warning message but continue with the startup process.

The script uses colorized console output and clear error messages to help developers understand and troubleshoot any issues that may arise during the startup.
