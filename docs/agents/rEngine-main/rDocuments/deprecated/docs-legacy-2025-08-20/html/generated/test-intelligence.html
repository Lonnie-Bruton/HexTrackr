test-intelligence.js Documentation

Purpose & Overview

This script is a test suite for the rEngineMCP Advanced Intelligence System. It simulates the loading and testing of the agent database, which powers the intelligent search capabilities of the rEngineMCP platform. The script is designed to validate the proper functioning of the intelligence system, ensuring that the agent database is loaded correctly and the search functionality works as expected.

Technical Architecture

The script consists of a single IntelligenceTest class, which encapsulates the logic for loading the agent database and performing intelligent searches. The class has the following key components:

  1. Data Properties:

    • functionMatrix: Stores the function category and metadata information.
    • errorPatterns: Stores the known error patterns and bug information.
    • projectMemory: Stores the project-level entities and metadata.
    • currentTasks: Stores the active project-level tasks and metadata.
    • agentDatabase: Stores the agent database, which contains the historical knowledge used for intelligent searches.

  2. Methods:

    • loadJSON(filePath): Utility function to load JSON data from a file path.
    • loadAgentIntelligence(): Responsible for loading the agent database files and initializing the data properties.
    • searchFunctionDatabase(query): Performs an intelligent search on the function database based on the provided query.
    • testIntelligentSearch(): Executes a series of predefined test queries and displays the search results.

The script also includes a runTest() function, which instantiates the IntelligenceTest class and runs the test suite.

Dependencies

This script depends on the following external modules:

  • fs-extra: Provides an enhanced file system API with additional functionality, such as readJson() for reading JSON files.
  • path: Provides utilities for working with file and directory paths.
  • url: Provides utilities for URL resolution and parsing.

Key Functions/Classes

IntelligenceTest Class

Constructor:

  • Initializes the data properties to null.

loadJSON(filePath):

  • Parameters:
    • filePath (string): The path to the JSON file to be loaded.
  • Returns:
    • The parsed JSON data or null if the file does not exist or failed to load.
  • Functionality:
    • Checks if the file exists at the provided path.
    • Reads the JSON file and returns the parsed data.
    • Logs an error message if the file fails to load.

loadAgentIntelligence():

  • Returns:
    • true if the agent database files were successfully loaded, false otherwise.
  • Functionality:
    • Logs the path to the agent memory directory.
    • Loads the following agent database files:
      • functions.json: Stores the function category and metadata information.
      • errors.json: Stores the known error patterns and bug information.
      • memory.json: Stores the project-level entities and metadata.
      • tasks.json: Stores the active project-level tasks and metadata.
      • github_copilot_memories.json: Stores the agent database, which contains the historical knowledge used for intelligent searches.
    • Logs the loading status of each database file.

searchFunctionDatabase(query):

  • Parameters:
    • query (string): The search query to be used for the intelligent search.
  • Returns:
    • An array of up to 5 relevant function search results, each containing the following properties:
      • name: The name of the function.
      • file: The file in which the function is located.
      • category: The category to which the function belongs.
      • description: The description of the function.
      • signature: The signature of the function.
      • line: The line number where the function is defined (or "unknown" if not available).
  • Functionality:
    • Checks if the functionMatrix data is available.
    • Splits the search query into individual terms.
    • Searches through all function categories and their associated functions, looking for matches based on the function name, description, file, and category.
    • Returns the top 5 matching results.

testIntelligentSearch():

  • Functionality:
    • Logs a message indicating the start of the intelligent search test.
    • Defines a set of test queries to be used for the search.
    • Iterates through the test queries, calling searchFunctionDatabase() for each one and logging the results.

runTest() Function

  • Functionality:
    • Instantiates the IntelligenceTest class.
    • Calls the loadAgentIntelligence() method to load the agent database.
    • If the database is loaded successfully, calls the testIntelligentSearch() method to run the test suite.
    • Logs an error message if the agent database fails to load.

Usage Examples

To run the test suite, simply execute the test-intelligence.js script:

node test-intelligence.js

This will execute the runTest() function, which will load the agent database and perform the intelligent search tests.

Configuration

This script does not require any external configuration. The file paths for the agent database files are hardcoded within the loadAgentIntelligence() method.

Error Handling

The script handles errors in the following ways:

  1. loadJSON(filePath):

    • If the file does not exist or fails to load, the method logs an error message and returns null.

  2. loadAgentIntelligence():

    • If any of the agent database files fail to load, the method logs an error message and returns false.

  3. searchFunctionDatabase(query):

    • If the functionMatrix data is not available, the method returns an empty array.

Integration

This script is designed to test the rEngineMCP Advanced Intelligence System, which is a core component of the rEngineMCP platform. The intelligent search capabilities provided by the agent database are crucial for various features and functionalities within the rEngineMCP system.

Development Notes

  • The script assumes that the agent database files are located in the rMemory/rAgentMemories directory relative to the script's location. This may need to be adjusted if the file structure changes.
  • The searchFunctionDatabase() method currently only returns the top 5 matching results. This can be modified to return more or less results as needed.
  • The test queries used in the testIntelligentSearch() method are hardcoded. These can be expanded or modified to suit the specific testing requirements.
