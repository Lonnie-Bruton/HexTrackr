Catalog Manager

This script defines the CatalogManager class, which manages catalog mappings between item serials and catalog IDs. It provides a robust solution for data integrity, validation, synchronization with inventory data, storage optimization, and future extensibility for multiple catalog providers. This class replaces the simpler catalogMap object, offering enhanced functionality and maintainability within the larger inventory management system.

How to Use

This script is designed to be used within a web browser environment. It does not run from the command line. The CatalogManager class is instantiated and made available globally via window.catalogManager. You interact with it through its methods.

Example:

// Get the catalog ID for serial '123' (assuming Numista is the provider)
const catalogId = window.catalogManager.getCatalogId('123');

// Set the catalog ID for serial '456' to '98765' (Numista)
window.catalogManager.setCatalogId('456', '98765');

// Synchronize an inventory item
const updatedItem = window.catalogManager.syncItem({ serial: '789', name: 'Coin',  });

// Remove orphaned mappings
const removedCount = window.catalogManager.removeOrphanedMappings(window.inventory);

// Get storage stats
const stats = window.catalogManager.getStorageStats();
console.log(stats); // Example: {key: "catalogMap", bytes: 1234, kb: 1.21, entries: 5}

Core Logic Breakdown

The CatalogManager class provides methods for managing catalog mappings:

  • constructor(options): Initializes the CatalogManager with configuration options like storageKey, saveCallback, and debug. It loads existing mappings from localStorage using the specified key.
  • _load(): Loads mappings from localStorage using the configured storageKey. Handles legacy data format for backward compatibility.
  • _save(): Saves mappings to localStorage and calls the optional saveCallback function if provided.
  • removeOrphanedMappings(currentInventory): Removes mappings for serials that are no longer present in the provided currentInventory array.
  • deduplicateMappings(): Currently a noop, intended for future optimization to deduplicate mappings across different serials.
  • getStorageStats(): Returns statistics about the storage used by the catalog mappings, including the key, size in bytes and kilobytes, and the number of entries.
  • validateCatalogId(catalogId, provider): Validates the format of a catalog ID based on the provider. Currently only supports Numista, where IDs must be numeric.
  • getCatalogId(serial, provider): Retrieves the catalog ID for a given serial and provider.
  • setCatalogId(serial, catalogId, provider): Sets the catalog ID for a given serial and provider. Validates the ID before storing.
  • getSerialsByCatalogId(catalogId, provider): Returns an array of serials associated with a specific catalog ID and provider.
  • syncItem(item): Synchronizes an individual inventory item with the catalog mapping, ensuring consistency between item.numistaId and the stored mapping.
  • syncInventory(items): Synchronizes an array of inventory items with their catalog mappings.
  • cleanupOrphans(inventory): Removes mappings for serials not present in the given inventory, similar to removeOrphanedMappings.
  • exportMappings(): Returns a copy of the internal _mappings object for backup purposes.
  • importMappings(mappings, merge): Imports mappings from a provided object. Optionally merges with existing data if merge is true.
  • getStats(): Returns summary statistics about the current mappings, including the total number of mappings and storage size.

The script initializes a global instance of CatalogManager named catalogManager and assigns it to window.catalogManager for global access. It also configures a saveCallback to update storage statistics if an updateStorageStats function exists.

Configuration & Dependencies

This script depends on the following:

  • localStorage: The browser's local storage API is used for persistent storage of catalog mappings.
  • CATALOG_MAP_KEY (global variable): Defines the key used for storing the catalog mappings in localStorage.
  • DEBUG (global variable): A boolean flag that controls debug logging.
  • saveData and loadDataSync (external functions): Assumed to be helper functions for interacting with localStorage; not defined within this script.
  • window.inventory (global variable, optional): Contains the inventory data, used by removeOrphanedMappings.

It does not have any external npm package dependencies.
