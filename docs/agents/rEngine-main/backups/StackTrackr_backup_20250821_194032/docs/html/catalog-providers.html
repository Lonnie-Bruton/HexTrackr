Catalog Providers

This script defines and manages catalog providers, which are used to interact with online coin catalogs. It establishes a registry of providers, allowing for easy access and retrieval of specific catalog information based on provider keys. The initial provider implemented is NumistaCatalogProvider, which interacts with the Numista catalog. This script lays the groundwork (Phase 1D) for integrating multiple online catalog resources into the application.

How to Use

This script is intended to be included as a JavaScript module within a larger web application. It does not run standalone from the command line. It exposes global variables NumistaCatalogProvider and CatalogProviders when used in a browser environment.

Example Usage (within a web application):

// Get the Numista catalog provider
const numistaProvider = CatalogProviders.get('numista');

// Normalize a Numista ID
const normalizedId = numistaProvider.normalizeId('N#12345'); // Returns '12345'

// Build a Numista link
const numistaLink = numistaProvider.buildLink('12345'); // Returns 'https://en.numista.com/catalogue/pieces12345.html'

// Check if an ID is valid for Numista
const isValid = numistaProvider.isValid('N#12345'); // Returns true

// Register a new provider (hypothetical example)
class MyCatalogProvider extends CatalogProvider {
  // ... implementation ...
}
CatalogProviders.register(new MyCatalogProvider());

// Get a list of all registered providers
const providers = CatalogProviders.list();

Core Logic Breakdown

  1. CatalogProvider Base Class: This base class (defined in catalog-api.js, not shown here) is assumed to exist and provides the foundation for all catalog providers.

  2. NumistaCatalogProvider Class: This class extends CatalogProvider and implements specific logic for interacting with the Numista catalog.

    • constructor(): Initializes the provider with the name "Numista" and the key "numista".
    • normalizeId(id): Cleans and normalizes a Numista ID by removing leading "N#" and trimming whitespace.
    • buildLink(id): Constructs a URL to a specific Numista catalog entry based on the provided ID.
    • isValid(id): Validates a Numista ID using regular expressions.

  3. CatalogProviders Module: This immediately invoked function expression (IIFE) creates a registry for catalog providers.

    • map: An object used to store registered providers, keyed by their key or lowercase name.
    • register(p): Registers a new provider in the map.
    • get(key): Retrieves a provider from the map based on its key. Defaults to 'numista' if the key is not found.
    • list(): Returns an array of all registered providers.
    • The IIFE immediately registers the NumistaCatalogProvider.

  4. Global Exposure: The last line of the script checks if it's running in a browser environment (typeof window!=='undefined') and, if so, exposes the NumistaCatalogProvider and CatalogProviders globally.

Configuration & Dependencies

This script depends on the CatalogProvider base class defined in a separate file, catalog-api.js. There are no external npm package dependencies or configuration files required. The script is self-configuring, with the NumistaCatalogProvider being registered automatically.
