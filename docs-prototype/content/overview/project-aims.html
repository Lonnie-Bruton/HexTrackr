Project Aims and Overview

High-Level Purpose

HexTrackr is a specialized, lightweight web application designed to serve two primary functions for cybersecurity professionals and teams:

  1. Vulnerability Management: To import, track, and visualize vulnerability data from security scanners.
  2. Ticket Management: To provide a simple, efficient system for creating and managing remediation tasks.

The core aim of HexTrackr is to bridge the gap between vulnerability identification and remediation tracking. It provides a centralized platform where security findings can be processed and converted into actionable tickets, streamlining the workflow from detection to resolution.

Key Features

Vulnerability Management (vulnerabilities.html)

  • CSV Data Import: Ingests vulnerability data from CSV exports, with built-in support for common scanner formats.
  • Modern Dashboard: Utilizes the Tabler.io framework to present a clean, data-rich dashboard.
  • Key Metrics Display: Shows at-a-glance statistics, including counts of critical, high, medium, and low severity vulnerabilities, and aggregates of VPR (Vulnerability Priority Rating) scores.
  • Advanced Data Grid: Employs AG Grid to display vulnerability data, offering powerful sorting, filtering, and searching capabilities.
  • Time-Series Tracking: The database schema is designed to track when vulnerabilities are first seen and last seen, allowing for trend analysis.

Ticket Management (tickets.html)

  • Simple CRUD Interface: Provides a straightforward interface for Creating, Reading, Updating, and Deleting remediation tickets.
  • Markdown Support: Allows for detailed descriptions and notes using Markdown, with a terminal-style viewer for clear rendering.
  • Status and Priority Tracking: Each ticket has a status (e.g., Open, In Progress, Closed) and a priority level, enabling effective workflow management.
  • CSV Import/Export: Supports bulk import and export of tickets, facilitating integration with other systems or offline analysis.

Shared Features

  • Unified Settings: A shared settings modal provides a consistent way to configure application settings across both the ticket and vulnerability pages.
  • Responsive Design: Both interfaces are fully responsive and adapt to various screen sizes.
  • Docker-Based Deployment: The entire application is containerized, ensuring easy, consistent, and reliable deployment in any environment.
  • Persistent Storage: Uses a SQLite database with a mounted volume in Docker to ensure all data is saved reliably.
