First Run Guide

This guide will walk you through launching the HexTrackr application for the first time and verifying that it is running correctly. This process assumes you have already completed the steps in the Installation Guide.

1. Start the Application

The HexTrackr application is managed entirely through Docker Compose, which simplifies the startup process.

From the root directory of the project, run the following command:

docker-compose up -d

What this command does:

  • It pulls the necessary Docker images if they are not already present on your system.
  • It builds the hextrackr service using the Dockerfile.node.
  • It starts the container in detached mode (-d), meaning it will run in the background.
  • It maps port 8080 on your local machine to port 8080 inside the container.
  • It creates and mounts a volume for the SQLite database (data/), ensuring your data persists even if the container is removed.

2. Verify the Container is Running

After running the up command, you can check the status of your Docker containers to ensure the application started successfully.

docker-compose ps

You should see output similar to the following, indicating that the hextrackr service is Up and running:

   Name                 Command               State           Ports         
----------------------------------------------------------------------------
hextrackr_app_1   npm start                     Up      0.0.0.0:8080->8080/tcp

3. Access the Application in Your Browser

Once the container is running, you can access the HexTrackr user interface. Open your web browser and navigate to one of the following URLs:

  • Ticket Management: http://localhost:8080/tickets.html
  • Vulnerability Management: http://localhost:8080/vulnerabilities.html

4. Initial State Verification

On the first run, the application will be in a clean state.

Database Initialization

  • The backend server automatically checks for the existence of the hextrackr.db file in the data/ directory.
  • Since this is the first run, the file will not exist. The server will automatically execute the scripts/init-database.js script to create the database and all necessary tables.
  • You can verify this by checking the data/ directory in your project folder; a hextrackr.db file should now be present.

User Interface

  • Tickets Page: The ticket table should be visible but empty. You should see the header, search bar, and the "Add Ticket" button.
  • Vulnerabilities Page: The dashboard will show all statistics as zero. The main data grid will be empty. You should see the "Import CSV" button.

5. Next Steps

The application is now running and ready for use. You can proceed to:

  • Manually add a new ticket using the "Add Ticket" button on the tickets page.
  • Import vulnerability data by clicking the "Import CSV" button on the vulnerabilities page.

If you encounter any issues, you can view the container's logs by running:

docker-compose logs -f app

This will show you the real-time output from the Node.js server, which can be helpful for debugging.
