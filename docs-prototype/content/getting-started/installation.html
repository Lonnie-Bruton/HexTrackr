Installation Guide

This guide provides instructions for installing and configuring the HexTrackr application. The entire application is containerized using Docker, which ensures a consistent and reliable setup process.

System Requirements

  • Docker: You must have Docker and Docker Compose installed on your server or local machine.
    • For server environments, follow the official Docker Engine installation guide for your Linux distribution.
    • For local setup, install Docker Desktop.
  • Git: A Git client is required to clone the application repository.

1. Clone the Repository

First, clone the HexTrackr repository to your desired location on the server or your local machine.

git clone <repository-url>
cd HexTrackr

Replace <repository-url> with the actual URL of the Git repository.

2. Configuration

The application is configured to run out-of-the-box with sensible defaults. No mandatory environment variables are required for the initial setup.

Port Configuration

  • By default, the application runs on port 8080.
  • If you need to run the application on a different port, you can modify the docker-compose.yml file. Change the ports mapping from "8080:8080" to "<your-port>:8080".

Example for running on port 8888:

services:
  app:
    # ...
    ports:
      - "8888:8080"
    # ...

3. Build and Launch the Application

The application stack is managed by Docker Compose. This single command will build the Docker image and start the application container.

From the root directory of the project, run:

docker-compose up -d --build

What this command does:

  • --build: Forces Docker to build the hextrackr image from the Dockerfile.node before starting the container. This is recommended for the first-time setup to ensure all dependencies are correctly installed.
  • -d: Runs the container in detached mode, so it continues to run in the background.

4. Database and Storage

  • Database: The application uses a SQLite database. The database file (hextrackr.db) is stored in the data/ directory.
  • Persistent Storage: The docker-compose.yml file defines a volume that maps the data/ directory on the host machine to the container. This ensures that all database information is preserved even if the container is stopped, removed, or updated.
  • Automatic Initialization: On the first launch, the application will automatically create the hextrackr.db file and set up the necessary tables.

5. Verify the Installation

To confirm that the application is running correctly, you can check the status of the Docker containers:

docker-compose ps

The output should show the hextrackr service with a state of Up.

You can now access the application by navigating to http://localhost:8080/tickets.html or http://localhost:8080/vulnerabilities.html in your web browser (or the custom port you configured).

6. Updating the Application

To update to a newer version of the application:

  1. Pull the latest changes from the Git repository: git pull
  2. Re-build and restart the container: docker-compose up -d --build

Docker will pull the latest base image, rebuild the application container with the new code, and restart it. Your data in the data/ directory will be preserved.
