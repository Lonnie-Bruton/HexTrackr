User Guide: Shared Settings Modal

The HexTrackr application features a unified settings modal that is shared across both the Ticket Management and Vulnerability Management pages. This provides a consistent and centralized location for all application-level configurations.

Accessing the Settings Modal

  1. Click the gear icon (<i class="fa-solid fa-cog"></i>) located in the main header of either the tickets or vulnerabilities page.
  2. The settings modal will appear as an overlay.

How It Works

The settings modal is a "shared component," meaning the same HTML and JavaScript code is used on all pages.

  • scripts/shared/settings-modal.js: This script contains all the logic for the modal, including fetching its HTML content, handling user interactions, and communicating with the backend API.
  • scripts/shared/settings-modal.html: This file contains the HTML structure of the modal itself.
  • Dynamic Injection: When a page loads, the settings-modal.js script automatically fetches the settings-modal.html content and injects it into the page. This ensures that any page can use the modal simply by including the script.

Features of the Settings Modal

The modal is organized into several tabs, each handling a different aspect of the application's configuration.

1. General Settings

  • Theme Customization: Allows you to switch between light and dark modes for the user interface.
  • API Configuration: Provides fields to enter API keys or other credentials needed for integrations (e.g., ServiceNow).

2. Data Management

  • Import Data: Provides a unified interface for importing CSV files for both tickets and vulnerabilities. This is the same functionality accessed via the "Import" buttons on the main pages.
  • Export Data: Allows you to export all tickets to a CSV file.
  • Clear Data:
    • Clear Tickets: Permanently deletes all records from the tickets table.
    • Clear Vulnerabilities: Permanently deletes all records from the vulnerabilities and vulnerability_imports tables.
    • A confirmation prompt is shown before any data is deleted to prevent accidental loss.

3. ServiceNow Integration

  • Configuration: Contains fields to set up the connection to a ServiceNow instance, including the instance URL, username, and password.
  • Test Connection: A button to verify that the provided credentials are correct and that the application can successfully connect to the ServiceNow API.
  • Ticket Creation: Once configured, this integration allows for the creation of ServiceNow tickets directly from vulnerabilities within HexTrackr.

4. About

  • Version Information: Displays the current version of the HexTrackr application.
  • Links: Provides links to project documentation, the Git repository, and other relevant resources.

Saving Settings

  • Most settings are saved automatically when changed.
  • For settings that require explicit confirmation (like API keys), a "Save" button is provided within the relevant tab.
  • All settings are stored in the backend database, ensuring they persist across sessions and are applied consistently throughout the application.
