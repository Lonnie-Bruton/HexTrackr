User Guide: Import and Export

HexTrackr provides functionality to bulk import and export data using the CSV (Comma-Separated Values) format. This is useful for migrating data from other systems, backing up your work, or performing offline analysis.

Importing Data

You can import data for both tickets and vulnerabilities. The import functionality is accessible via buttons on their respective pages.

Importing Vulnerabilities

  1. Navigate to the Vulnerability Management page (vulnerabilities.html).
  2. Click the "Import CSV" button located in the header.
  3. A modal will appear, prompting you to select a CSV file and specify the vendor/scanner format (e.g., Tenable, Qualys).
  4. Choose your file and click "Import".

Backend Process (POST /api/vulnerabilities/import):

  • The backend receives the file and uses the multer middleware to handle the upload.
  • The papaparse library reads and parses the CSV data.
  • An import record is created in the vulnerability_imports table to log the transaction.
  • Each row in the CSV is processed and inserted as a new record into the vulnerabilities table. The server is designed to handle large files efficiently.

Importing Tickets

  1. Navigate to the Ticket Management page (tickets.html).
  2. Click the "Import CSV" button.
  3. Select the CSV file containing your ticket data.
  4. Click "Import".

Backend Process (POST /api/tickets/import):

  • The process is similar to vulnerability imports.
  • The server parses the CSV file.
  • Each row is validated and inserted into the tickets table.

Required CSV Headers for Tickets:
To import tickets successfully, your CSV file must contain the following headers:
title, status, priority, description

Exporting Data

Currently, data export is available for tickets.

Exporting Tickets

  1. Navigate to the Ticket Management page (tickets.html).
  2. Click the "Export CSV" button.
  3. Your browser will automatically download a file named tickets.csv.

Backend Process (GET /api/tickets/export):

  • The server queries the tickets table to fetch all records.
  • It uses the papaparse library to convert the JSON data from the database into a CSV-formatted string.
  • It sets the appropriate HTTP headers (Content-Type: text/csv and Content-Disposition) to instruct the browser to download the file.

This provides a quick and easy way to get a complete backup of all your ticket data.
