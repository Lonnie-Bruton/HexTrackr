Project Dependencies

This document lists all the production and development dependencies used in the HexTrackr project, as defined in the package.json file.

Production Dependencies (dependencies)

These packages are required for the application to run in a production environment.

Package Version Description
@google/generative-ai ^0.24.1 Provides integration with Google's generative AI models, likely for future or experimental features.
compression ^1.7.4 A Node.js middleware that compresses response bodies to improve performance.
cors ^2.8.5 A Node.js middleware to enable Cross-Origin Resource Sharing (CORS).
dotenv ^17.2.1 Loads environment variables from a .env file into process.env.
express ^4.18.2 The core web application framework for Node.js, used to build the server and API.
jszip ^3.10.1 A library for creating, reading, and editing .zip files. Used for data export features.
marked ^16.2.1 A high-performance Markdown parser and compiler. Used to render Markdown descriptions in the UI.
multer ^2.0.2 A Node.js middleware for handling multipart/form-data, primarily used for file uploads (e.g., CSV imports).
papaparse ^5.4.1 A powerful and fast CSV parser for the browser and Node.js. Used to process imported CSV files.
playwright ^1.55.0 A framework for web testing and automation. May be used for automated report generation or testing.
sqlite3 ^5.1.6 The Node.js driver for the SQLite3 database, enabling database interaction.

Development Dependencies (devDependencies)

These packages are only needed for local development and testing. They are not installed in the production environment.

Package Version Description
eslint ^9.34.0 A pluggable and configurable linter tool for identifying and reporting on patterns in JavaScript. Used to enforce code quality.
nodemon ^3.0.1 A utility that monitors for any changes in your source and automatically restarts the server. Used for the npm run dev script.
