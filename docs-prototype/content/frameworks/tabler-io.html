Tabler.io Framework

The HexTrackr application uses the Tabler.io UI kit for the Vulnerability Management page (vulnerabilities.html). Tabler.io is a premium and open-source dashboard template built on Bootstrap, providing a clean, modern, and responsive interface for data-rich applications.

Integration

Tabler.io is integrated into the project via a CDN link in the <head> section of vulnerabilities.html.

<link href="https://cdn.jsdelivr.net/npm/@tabler/core@1.0.0-beta17/dist/css/tabler.min.css" rel="stylesheet">

This single line imports the core Tabler CSS, which includes styles for layout, components, and typography.

Key Components Used

The vulnerabilities.html page is built almost entirely with Tabler.io components, creating a sophisticated and consistent user experience.

1. Page Layout

  • <div class="page"> and <div class="page-wrapper">: These are the main containers that structure the entire page, providing the foundational layout for the dashboard.
  • <header class="navbar">: The shared application header is built using Tabler's navbar component, ensuring a consistent look and feel with the rest of the UI.
  • <div class="page-body">: This container holds the main content of the page, including statistics cards and the data grid.

2. Cards (<div class="card">)

  • Stats Cards: The key statistics at the top of the page (e.g., "Critical Vulnerabilities," "High Severity VPR") are displayed in Tabler cards. These cards use a combination of icons, titles, and metric values to present information clearly.
  • Main Content Card: The AG Grid for vulnerabilities is enclosed within a large card that provides a clear visual boundary for the main interactive element of the page.

3. Data Visualization

  • Charts: The page is designed to include charts (e.g., for vulnerability trends). While Chart.js is used for the charting library itself, the charts are embedded within Tabler cards to seamlessly integrate with the dashboard's aesthetic.

4. Forms and Controls

  • Buttons (<a href="#" class="btn btn-primary">): Action buttons like "Import CSV" are styled using Tabler's button classes.
  • Input Groups: The search and filter controls for the data grid are styled using Tabler's form components for a cohesive look.

5. Icons

  • Font Awesome: Tabler.io is designed to work well with icon libraries. HexTrackr uses Font Awesome, and the icons are integrated into Tabler components like buttons and stat cards to provide visual cues.

Customization and Responsiveness

  • Responsive Design: Tabler.io is inherently responsive. The layout, cards, and other components automatically adjust to different screen sizes, from large desktops to mobile devices.
  • Custom Styles: While Tabler provides a comprehensive set of styles, custom CSS in styles/pages/vulnerabilities.css is used to fine-tune the appearance. For example, custom styles are applied to constrain the maximum width of the page on ultrawide monitors, ensuring the content remains centered and readable.
  • AG Grid Integration: The AG Grid component is styled with the ag-theme-alpine theme, but it is embedded within Tabler cards and layouts, making it a central part of the Tabler-based dashboard.
