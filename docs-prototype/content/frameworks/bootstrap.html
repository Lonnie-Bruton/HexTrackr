Bootstrap Framework

The HexTrackr application utilizes the Bootstrap 5 framework, primarily for the user interface of the Ticket Management page (tickets.html). Bootstrap provides a responsive, mobile-first grid system and a comprehensive set of pre-styled components that enable rapid and consistent UI development.

Integration

Bootstrap is integrated into the project via a CDN link in the <head> section of tickets.html.

<link href="https://cdn.jsdelivr.net/npm/@tabler/core@1.0.0-beta17/dist/css/tabler.min.css" rel="stylesheet">

Note: While the link references Tabler, Bootstrap components are used throughout tickets.html. The project uses a mix of both frameworks across different pages.

Key Components Used

The tickets.html page leverages several core Bootstrap components to build its interface:

1. Layout and Grid System

  • Containers (<div class="container-fluid">): The main content is wrapped in a fluid container to ensure it spans the full width of the viewport.
  • Grid System (<div class="row">, <div class="col-md-*">): The layout is structured using Bootstrap's responsive grid to organize elements like the main table, search bar, and action buttons. This ensures the layout adapts cleanly to different screen sizes.

2. Cards (<div class="card">)

  • The main data table and its surrounding elements (like the header and pagination) are enclosed within a Bootstrap card, giving the section a clear, bordered appearance.

3. Forms and Inputs (<div class="form-group">, <input class="form-control">)

  • The search bar and the "Add Ticket" modal use Bootstrap's form styling for a clean and consistent look. This includes input fields, labels, and buttons.

4. Buttons (<button class="btn btn-*">)

  • Action buttons such as "Add Ticket," "Import CSV," "Save," and "Delete" are styled using Bootstrap's button classes (btn-primary, btn-danger, etc.).

5. Modals (<div class="modal">)

  • The "Add/Edit Ticket" and "Delete Confirmation" functionalities are implemented using Bootstrap modals. These provide a clean overlay for user interaction without navigating away from the main page.

6. Tables (<table class="table table-striped">)

  • The list of tickets is displayed in a Bootstrap-styled table. The .table-striped class is used to add zebra-striping to table rows, which improves readability.

7. Navigation

  • While the primary header is a shared component, its structure is compatible with Bootstrap's layout principles, ensuring it integrates smoothly with the page content.

Customization

While HexTrackr relies on Bootstrap's default styles for many components, custom CSS is added in styles/pages/tickets.css to override or extend the framework's styling to match the application's specific design requirements. This hybrid approach allows for rapid development while maintaining a unique brand identity.
