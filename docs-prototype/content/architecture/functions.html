HexTrackr Core Functions

This document details the core backend functions defined in server.js.

1. initDb()

  • Purpose: Initializes the SQLite database.
  • Description: This function checks if the database file (data/hextrackr.db) exists. If not, it runs the scripts/init-database.js script to create the database and its tables. It also ensures that all necessary columns exist in the vulnerabilities table by adding them if they are missing, which supports backward compatibility for existing databases.

2. API Route Handlers

The following functions handle the API requests for different resources.

2.1. Vulnerability Endpoints

  • GET /api/vulnerabilities/stats: Fetches statistics about vulnerabilities, grouped by severity.
  • GET /api/vulnerabilities/trends: Retrieves historical vulnerability data for the last 14 days to populate trend charts.
  • GET /api/vulnerabilities: Gets a paginated list of vulnerabilities, with support for searching and filtering by severity.
  • POST /api/vulnerabilities/import: Imports vulnerabilities from an uploaded CSV file. It parses the file, creates an import record, and inserts the vulnerability data into the database.
  • DELETE /api/vulnerabilities/clear: Deletes all vulnerability data from the database.
  • GET /api/imports: Retrieves the history of all CSV imports.

2.2. Ticket Endpoints

  • GET /api/tickets: Fetches all tickets from the database.
  • POST /api/tickets: Creates a new ticket.
  • PUT /api/tickets/:id: Updates an existing ticket.
  • DELETE /api/tickets/:id: Deletes a ticket.
  • POST /api/tickets/migrate: Migrates tickets from a JSON object, used for data restoration.
  • POST /api/import/tickets: Imports tickets from a JSON object, typically from a CSV file parsed on the frontend.

2.3. Backup and Restore Endpoints

  • GET /api/backup/stats: Provides statistics on the number of tickets and vulnerabilities, and the database size.
  • GET /api/backup/vulnerabilities: Exports all vulnerabilities as a JSON object.
  • GET /api/backup/tickets: Exports all tickets as a JSON object.
  • GET /api/backup/all: Exports all tickets and vulnerabilities in a single JSON object.
  • DELETE /api/backup/clear/:type: Clears all data for a specified type (vulnerabilities, tickets, or all).
  • POST /api/restore: Restores data from an uploaded backup file.

2.4. Site and Location Endpoints

  • GET /api/sites: Fetches all sites.
  • GET /api/locations: Fetches all locations.
