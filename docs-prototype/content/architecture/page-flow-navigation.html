HexTrackr Page Flow and Navigation

This document describes the user navigation and page flow within the HexTrackr application.

1. Main Pages

The application consists of two primary pages:

  • Ticket Management: tickets.html
  • Vulnerability Management: vulnerabilities.html

A shared header provides consistent navigation between these two pages.

2. Ticket Management (tickets.html)

2.1. Initial View

  • The page loads with a table displaying all existing tickets.
  • Statistics cards show a summary of ticket statuses (Total, Open, Overdue, Completed).
  • Users can search, filter, and sort the ticket data.

2.2. User Actions

  • Add New Ticket: Opens a modal (#ticketModal) to create a new ticket.
  • View Ticket: Opens a modal (#viewTicketModal) with a detailed, read-only view of the selected ticket.
  • Edit Ticket: From the view modal, users can open the edit modal, which is the same as the "Add New Ticket" modal but pre-filled with the ticket's data.
  • Delete Ticket: A button in the actions column of the ticket table allows for the deletion of a ticket.
  • Export Data: Users can export the ticket data in various formats (CSV, Excel, JSON, PDF, HTML).
  • Attach Documentation: Allows users to attach files to a ticket.

3. Vulnerability Management (vulnerabilities.html)

3.1. Initial View

  • The page loads with a dashboard view, including:
    • Statistics cards for vulnerabilities by severity.
    • A historical trend chart.
    • A data workspace with three views: Table, Devices, and Vulnerabilities.
  • The default view is the Table View, which uses AG Grid to display all vulnerabilities.

3.2. User Actions

  • Switch Views: Users can switch between the Table, Devices, and Vulnerabilities views.
  • Search and Filter: Users can search for specific vulnerabilities and filter by severity.
  • View Device Details: In the Devices View, clicking on a device card opens a modal (#deviceModal) with a detailed security overview for that device.
  • View Vulnerability Details: In the Vulnerabilities View, clicking on a vulnerability card opens a modal (#vulnerabilityModal) with detailed information about that vulnerability.
  • Import Data: Users can import vulnerability data from a CSV file through the settings modal.
  • Export Data: Users can export data from the AG Grid or from the device/vulnerability detail modals.

4. Shared Components

4.1. Header

  • Provides navigation links to the "Tickets" and "Vulnerabilities" pages.
  • Includes a "Settings" button that opens the settings modal.

4.2. Settings Modal (#settingsModal)

  • Accessible from both main pages.
  • Provides functionality for:
    • Importing CSV data (tickets or vulnerabilities).
    • Backing up data (tickets, vulnerabilities, or all).
    • Restoring data from a backup.
    • Clearing application data.
