HexTrackr JavaScript Reference

This document provides an overview of the JavaScript code organization in HexTrackr.

1. Modular Architecture

The JavaScript code is structured into two main directories to promote a clean and maintainable codebase:

  • scripts/pages/: Contains JavaScript files specific to a single page (e.g., tickets.js, vulnerabilities.js).
  • scripts/shared/: Contains JavaScript files that provide functionality across multiple pages (e.g., settings-modal.js).

2. Page-Specific Scripts

Each main HTML file (tickets.html, vulnerabilities.html) has a corresponding JavaScript file in scripts/pages/. These files are responsible for:

  • Handling user interactions on that specific page.
  • Fetching and displaying data from the API.
  • Managing the state of the page.

2.1. tickets.js

  • Manages the ticket management interface.
  • Handles creating, reading, updating, and deleting tickets.
  • Interacts with the /api/tickets endpoints.

2.2. vulnerabilities.js

  • Manages the vulnerability management interface.
  • Handles fetching, filtering, and displaying vulnerability data.
  • Interacts with the /api/vulnerabilities endpoints.
  • Manages CSV imports and data visualization.

3. Shared Scripts

Shared scripts provide common functionality that can be reused across the application.

3.1. settings-modal.js

  • Controls the behavior of the settings modal.
  • Handles data backup, restore, and clearing operations.
  • Interacts with the /api/backup and /api/restore endpoints.

4. Script Loading

Scripts are loaded in the HTML files in the following order:

  1. Shared scripts (e.g., settings-modal.js)
  2. Page-specific scripts (e.g., tickets.js)

This ensures that shared functionality is available before page-specific code that may depend on it is executed.

<!-- Example from tickets.html -->
<script src="scripts/shared/settings-modal.js"></script>
<script src="scripts/pages/tickets.js"></script>

5. Key Libraries

  • jQuery: Used for DOM manipulation and event handling.
  • Bootstrap: Provides UI components and JavaScript plugins for the tickets page.
  • Tabler.io: Provides UI components and JavaScript plugins for the vulnerabilities page.
  • PapaParse: Used for parsing CSV files on the client and server.
  • Chart.js: Used for creating charts and graphs for vulnerability trends.
