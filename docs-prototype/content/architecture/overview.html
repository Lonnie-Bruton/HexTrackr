HexTrackr System Architecture

1. Overview

HexTrackr is a web-based application designed for managing cybersecurity tickets and vulnerabilities. It is built with a Node.js backend, an Express.js framework, and a SQLite database. The application is containerized using Docker for consistent deployment and scalability.

2. Core Components

2.1. Backend

  • Framework: Express.js
  • Language: Node.js
  • Database: SQLite 3
  • API: RESTful API for all data operations.

2.2. Frontend

  • Tickets Interface: tickets.html (Built with Bootstrap 5)
  • Vulnerabilities Interface: vulnerabilities.html (Built with Tabler.io)
  • JavaScript: Modular, with shared components and page-specific scripts.

2.3. Data Storage

  • Primary Database: A single SQLite database file (data/hextrackr.db) stores all application data.
  • File Uploads: Temporary storage in the uploads/ directory for processing.

3. Deployment

  • Containerization: The application is designed to run exclusively within a Docker container.
  • Orchestration: docker-compose.yml defines the service, network, and volume configurations.
  • Service: A single hextrackr service runs the Node.js application.
  • Port: The application is accessible on port 8080.

4. Application Flow

  1. The user accesses the application through tickets.html or vulnerabilities.html.
  2. The frontend makes API calls to the Node.js/Express backend.
  3. The backend processes requests, interacts with the SQLite database, and returns data in JSON format.
  4. Data is dynamically rendered on the frontend.

5. Key Architectural Patterns

  • Database-First: The application relies on the SQLite database as the single source of truth.
  • Modular JavaScript: Code is organized into shared/ and pages/ directories to promote reusability and separation of concerns.
  • RESTful API: A comprehensive set of API endpoints provides CRUD (Create, Read, Update, Delete) functionality for all data entities.
  • Stateless Backend: The backend is largely stateless, with all persistent data stored in the database.
