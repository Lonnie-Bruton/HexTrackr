Database Architecture

HexTrackr uses a database-first approach with a fallback to localStorage.

Primary Storage

  • Database: SQLite (data/hextrackr.db)
  • Rationale: SQLite provides a simple, file-based, and self-contained database solution that is perfect for a lightweight application like HexTrackr. It requires no separate server process.

Data Model

  • Tickets: Stores information about support tickets.
  • Vulnerabilities: Stores vulnerability data imported from CSVs. The data model is designed for time-series tracking, allowing for analysis of trends.
  • UPSERT Logic: When importing vulnerability data, the system performs an UPSERT (update or insert) to avoid creating duplicate entries.

Fallback Storage

  • localStorage: If the backend database is unavailable, the application can fall back to using the browser's localStorage for basic functionality. This is not a primary feature but provides resilience.
