Project Roadmap

This document provides a high-level overview of the HexTrackr project's strategic roadmap, current status, and future plans. It consolidates information from several planning documents to provide a single source of truth for the project's direction.

1. Strategic Roadmap (ROADMAP.md)

This is the primary document outlining the long-term vision and major feature releases for HexTrackr.

Completed Features

  • Core Infrastructure: Dual-app architecture, time-series database, modern UI with AG Grid, and a modular JavaScript pattern.
  • Chart & Visualization System: ApexCharts integration with real-time updates and responsive design.
  • Cisco API Integration: Secure OAuth2 authentication and vulnerability synchronization.

In Progress

  • Code Quality & Security Compliance: A critical, release-blocking initiative to resolve all security vulnerabilities identified by Codacy and reduce the total number of issues.
  • Ticket Modal Enhancement Suite: A series of high-priority improvements to the ticket creation and management modal.
  • Chart Timeline Enhancement: Extending the vulnerability trend chart to provide a more comprehensive timeline.

2. UI/UX Roadmap (UI_UX_ROADMAP.md)

This document focuses on the technical implementation details of the user interface and user experience enhancements.

Current Focus

  • Chart Timeline Extension: The immediate priority is to extend the vulnerability chart's timeline to the current date, providing a more continuous and informative visualization.

Completed UI/UX Achievements

  • Modular JavaScript Architecture: Successfully refactored the frontend code into a modular pattern with shared components and page-specific scripts.
  • Responsive Layout: Implemented a comprehensive responsive design for the AG Grid and other UI components, ensuring the application is usable on both desktop and mobile devices.
  • Settings Modal Unification: Created a single, unified settings modal that functions consistently across both the tickets and vulnerabilities pages.

3. Current Sprint Status (CURRENT_STATUS.md)

This document provides a snapshot of the current sprint's focus, priorities, and recent accomplishments.

Current Focus

  • Code Quality & Security Compliance: The team is actively working on fixing four critical security vulnerabilities identified by Codacy, which is a blocker for the next release (v1.0.3).
  • Ticket Modal Enhancements: In parallel, work is proceeding on improving the ticket modal, with the current task being the implementation of a reverse sort toggle.

Recent Accomplishments

  • AI Documentation Generation System: A comprehensive, automated documentation generation system has been successfully implemented, creating a live documentation portal with professional styling and content derived directly from the codebase.

4. Codacy Compliance Tracking (CODACY_COMPLIANCE_TRACKING.md)

This document tracks the project's progress toward meeting its code quality and security goals.

Current Status

  • 83 Total Issues: Down from over 370 after recent ESLint configuration improvements.
  • 4 Critical Security Issues: These are release-blocking and are the highest priority for the development team. The issues include a generic object injection sink and unsafe file system operations.

Release Criteria

  • Zero Critical/High Security Vulnerabilities.
  • Fewer than 50 total Codacy issues.
  • All security fixes must be tested and verified.
